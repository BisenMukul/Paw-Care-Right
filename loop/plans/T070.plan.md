## Key decisions (checker: scrutinize)
1. **Traversal guard = a shared boolean helper in `photos.constants.ts`, NOT a strict uuid regex.** The current guard is `key.startsWith(originalKeyPrefix(petId))` in both `confirmUpload` and `viewUrls`. A key like `pets/<petId>/original/../../<otherPet>/main/x.jpg` passes `startsWith` yet escapes the namespace once `deriveThumbKey`/`deriveMainKey` sign it → a cross-tenant READ primitive (harmless on literal-key S3 today, but must be closed now that `viewUrls` presigns GETs). The new helper `isKeyInPetNamespace(petId, key)` keeps the `startsWith` check AND additionally requires the **post-prefix remainder** to be a single flat segment: non-empty, contains **no `/`**, and contains **no `..`**. A strict `[0-9a-f-]+\.(jpg|png|webp)` regex is **rejected** as the design — the existing e2e uses the legitimate-but-non-uuid key `${originalKeyPrefix(petId)}never-uploaded.jpg` and expects **404** (object-not-found), so a strict regex would false-positive it to 400 and break a passing test. "No slash, no `..` in the remainder" closes traversal while leaving every legitimate flat `<uuid>.<ext>` (and `never-uploaded.jpg`) valid. `buildOriginalKey` only ever emits flat single-segment keys, so this rejects nothing real.
2. **Both call sites and the DTO stay as-is; only the guard body changes.** `confirmUpload` and `viewUrls` call `isKeyInPetNamespace(...)` and throw the same `BadRequestException` (same message tone) on false. HTTP contract unchanged: foreign-namespace and traversal keys both → `400 VALIDATION_FAILED`; well-formed-but-absent → `404`.
3. **Log factories live in a new `apps/api/test/factories/health-logs.ts`, re-exported by the `factories/index.ts` barrel.** Consumers keep importing from `./factories`. Deliverables: pure per-kind `logValue.*` builders, a `createHealthLog(prisma, petId, …)` DB factory, and `seedCompletedMedication(prisma, petId, …)` (Reminder + DONE ReminderEvent — the MED_GIVEN projection source). Only **clear** duplication is refactored: the ~20-line Reminder+ReminderEvent seed block duplicated across `filter_by_each_kind` and `vet-summary`, and the direct CHECK_REF `healthLog.create`. The real-endpoint helpers `validLogBody`/`createLog` (they exercise the POST path — intended coverage, not duplication) and the service-spec mock-row builders stay untouched.
4. **Coverage is measured, not assumed.** Jest `collectCoverageFrom: ["src/**/*.service.ts"]`, global thresholds statements/lines/functions ≥80, branches ≥70 (already in `apps/api/package.json`). Executor runs `pnpm --filter api test:cov` FIRST, reads the uncovered lines for the two Phase-6 services, and writes targeted unit tests only for **actual** gaps (candidates in §Tests). No threshold gaming (no blanket "call every method once"): each added test asserts real behaviour.
5. **The render-count straggler is a NEW test, not a mutation of the existing one.** T067's append test keeps its `photoKeys: []` fixtures (leave it green, unchanged). A sibling test gives page-1 item A a non-empty `photoKeys`, mounting the real `TimelinePhotoStrip` (separately memoized, owns the `usePhotoViewUrls` query already mocked at the `pet-photos-api` boundary in this file), and asserts `getKindDisplay` is invoked for A exactly once on page 1 and **never again** after appending page 2 (only C, D) — proving a photo-bearing row survives an append without re-rendering its `TimelineRow`.
6. **Test-race audit is read-only.** The Phase-6 mobile tests already await every render/fireEvent (RNTL v14). Executor does a quick read pass for assertions-outside-`waitFor`; if a genuine race exists it is journalled, but passing tests are **not** churned to chase hypothetical flakiness (S-size, "don't churn passing tests"). The only mobile test edited is `timeline-screen.test.tsx` (decision 5).

### Modify — API source
- `apps/api/src/photos/photos.constants.ts` — add and export:
  ```ts
  /** True iff `key` is a single flat object under this pet's original-upload
   *  namespace. Rejects a foreign prefix, path-traversal (`..`) and any extra
   *  `/` segment, so a derived thumb/main key can never escape the namespace. */
  export function isKeyInPetNamespace(petId: string, key: string): boolean {
    const prefix = originalKeyPrefix(petId);
    if (!key.startsWith(prefix)) return false;
    const remainder = key.slice(prefix.length);
    if (remainder.length === 0) return false;
    if (remainder.includes("/")) return false;
    if (remainder.includes("..")) return false;
    return true;
  }
  ```
  (No behaviour change to existing exports.)
