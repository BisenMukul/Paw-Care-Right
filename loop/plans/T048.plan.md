# Plan — T048: Result screen

## Objective (from card)
Build the mobile symptom-check **Result screen** at the T047 contract route
`/check/result/[checkId]`: urgency banner (tier color system) · summary · possible
causes · red-flags-to-watch · home care · do-nots · vet questions · a shared
**non-dismissible `<VetDisclaimer/>`** · actions (Find vet nearby maps deep link with
an "emergency vet near me" query for the top tiers, share-as-text, done→timeline).
`FALLBACK` status renders a distinct safe screen. This is **the phase's primary
PRODUCT_SPEC §5 mobile surface** — the disclaimer, the fail-upward framing, and the
emergency reinforcement are the safety-critical deliverables.

Deps satisfied in repo: T047 (`useCheck` in `apps/mobile/src/api/checks-api.ts`;
waiting screen already `router.replace`s to `/check/result/[checkId]` with **only**
`{ checkId }`), `packages/types` (`TriageResult`, `URGENCY_TIERS`, `URGENCY_SEVERITY`,
`SAFE_FALLBACK`, `CheckResponse`, `isTerminalCheckStatus`), `@pawcareright/config`
(`APP_DISPLAY_NAME`), react-native `Share` precedent (`app/family.tsx`), `expo-linking`
(in `apps/mobile/package.json`).

## Safety statement (PRODUCT_SPEC §5 / CLAUDE §7 — this IS the §5 mobile surface)

- **Disclaimer, non-dismissible, on every content state (§5 rule 1 / §7 rule 3).**
  A single shared `<VetDisclaimer/>` (plain `View`, **no** `Pressable`/close/onPress
  affordance) renders on all six required content states (5 tiers + FALLBACK) and on
  the redFlag variant. Every snapshot asserts it two ways (belt+braces): the snapshot
  content itself **and** an explicit `getByTestId("vet-disclaimer")` query. The
  `loading`/`error` non-content states show a spinner/retry only (no AI content, so no
  disclaimer) per decision D1.
- **Diagnosis-word ban — decision D2 (the ONE significant design decision).** CLAUDE §7
  rule 1 + the codebase norm (`packages/ai/src/triage/system-prompt.ts` keeps the token
  `"diagnos"` to the *single* never-use instruction; `packages/types/triage.ts`
  mechanically rejects `/diagnos/i` in every user-facing triage field) mean **no shipping
  user-facing string in this task may contain `diagnos*`.** SPEC §5 rule 1's *illustrative*
  disclaimer wording contains "diagnosis", but §5's binding requirement is the *content*
  (persistent, non-dismissible, "general guidance, not veterinary treatment; always
  consult a licensed veterinarian") and it explicitly notes the copy is "Localized once
  i18n lands" — i.e. the exact English tokens are not immutable. The strictest defensible
  reading that satisfies BOTH: write a disclaimer that is **at least as strong** while
  dropping the `/diagnos/` token. Pinned copy uses **"not veterinary care or treatment"**
  ("care" ⊇ "diagnosis", so this is broader/stronger, never weaker) and keeps SPEC's
  verbatim "Always consult a licensed veterinarian." The possible-causes section heading
  is **"What this could be"** — the §7 rule-1 sanctioned replacement phrasing — never
  "diagnosis". This preserves the §5 surface (present + non-dismissible + every result +
  consult-a-vet) and is documented under Risks for the checker.
- **Fail upward (§5 rule 2 / §7 rule 5).** `FALLBACK` status renders a distinct safe
  screen framed "we couldn't fully assess — check with a vet" over the frozen
  `SAFE_FALLBACK` content. A defensive guard also treats a terminal check whose `result`
  is missing as fallback (renders `SAFE_FALLBACK`), never a crash or a fabricated result.
  A load **error** shows a safe retry screen that still nudges toward a vet — never a
  silent retry-and-guess.
- **Emergency reinforcement (§5 rule 3 / §7 rule 4).** A check that carries `redFlag`
  renders a prominent emergency notice + a "See emergency steps" link to the T049
  interstitial route `/check/emergency/[checkId]`, placed **above** all AI content. This
  reinforces (never buries) the emergency; the interstitial itself remains T049's
  full-screen takeover. (Per the T047 flow a redFlag check reaches this screen only via
  T049 "continue" or T050 detail-reuse, never via the polling path.)
- **No dosing / no harm (§5 rule 4/5).** No new string names a drug, dose, or treatment;
  the `doNot` list is rendered as-authored from the server-validated result. All copy is
  externalized in `strings.ts` (§6); no hardcoded display name (§1a — injected from
  `APP_DISPLAY_NAME`).
- Nothing here weakens a §5 surface (it *builds* one), so the task is plannable, not
  SAFETY-ESCALATION.

## Key decisions (numbered; scrutinize in Risks)

- **D1 — Disclaimer on content states only; loading = spinner.** `<VetDisclaimer/>`
  renders on the 5 tier states + FALLBACK + redFlag variant. `loading`/`error` show
  spinner/retry (no AI content → no disclaimer required). Matches the card AC scope
  (per-tier + FALLBACK snapshots) and the §5 intent (disclaimer accompanies AI output).
- **D2 — Diagnosis-word decision (see Safety statement).** Disclaimer avoids `/diagnos/`
  via "not veterinary care or treatment"; causes heading "What this could be".
- **D3 — Trust the server-validated result shape; no client `parseTriage` on render.**
  Per T047 D3 the client types `useCheck` data as `CheckResponse`; the persisted result
  was schema-gated server-side (T034). The screen keys the safe screen on
  `status === "FALLBACK" || result === undefined` (a status/defensive-presence check),
  **not** on re-validating the triage body. Re-running `parseTriage` on the client is
  deliberately out of scope (avoids schema-drift false-fallbacks; server is the gate).
- **D4 — Tier banner is a data map, not per-tier components.** `URGENCY_DISPLAY:
  Record<Urgency, {containerClass,textClass,testId}>` drives one banner component; the
  label text comes from `strings.check.result.tierLabel[urgency]`. Colors: EMERGENCY_NOW
  red, VET_24H orange, VET_SOON amber, MONITOR blue, REASSURE green (exact NativeWind
  classes pinned below; default Tailwind palette — `text-red-600` is already used in the
  app, so the default color scale resolves under the shared preset).
- **D5 — FALLBACK = same renderer + a FALLBACK notice section (distinct snapshot).** The
  FALLBACK screen reuses the standard section renderer over `SAFE_FALLBACK` (VET_SOON,
  low, empty lists, the neutral no-human-medication `doNot`) plus a pinned notice banner
  → snapshot is distinct from a normal VET_SOON DONE result (notice present + empty
  lists). No separate screen component.
- **D6 — Find-vet uses one platform-agnostic maps deep link.** Pure `buildVetSearchUrl`
  builds `https://www.google.com/maps/search/?api=1&query=<encoded>` (Google's documented
  universal URL — opens the native Maps app on iOS/Android, browser fallback). Query is
  `"emergency vet near me"` for the top tiers (EMERGENCY_NOW, VET_24H;
  `URGENCY_SEVERITY[urgency] <= URGENCY_SEVERITY.VET_24H`) else `"veterinarian near me"`.
  Opened via `expo-linking` `openURL`. Chosen over `Platform.select` geo:/maps: branching
  so the builder is pure + deterministically unit-testable.
- **D7 — done → the existing Timeline tab.** `router.replace("/(tabs)/timeline")` — the
  honest, already-existing target (`app/(tabs)/timeline.tsx`); the card literally says
  "done→timeline". The result screen only receives `{ checkId }` (T047), not `petId`, so
  navigating to the pet-scoped Timeline tab (active-pet driven) is the correct target.
  T050 fills the tab's content; the route exists today.
- **D8 — Reuse `useCheck` (no new hook).** The check is terminal, so
  `checkRefetchInterval` returns `false` → a single fetch, no polling. If a non-terminal
  status somehow lands here, the screen falls through to the loading view (safe: shows no
  AI content).
- **D9 — Share via react-native `Share.share`** (precedent: `app/family.tsx`); payload
  built by pure `buildSharePayload`, which **always** appends the disclaimer line.

### `vet-search.ts`
```ts
import { URGENCY_SEVERITY, type Urgency } from "@pawcareright/types";
export const VET_SEARCH_MAPS_BASE = "https://www.google.com/maps/search/?api=1&query=";
export const EMERGENCY_VET_QUERY = "emergency vet near me";
export const ROUTINE_VET_QUERY = "veterinarian near me";
export function buildVetSearchQuery(urgency: Urgency): string {
  return URGENCY_SEVERITY[urgency] <= URGENCY_SEVERITY.VET_24H ? EMERGENCY_VET_QUERY : ROUTINE_VET_QUERY;
}
export function buildVetSearchUrl(urgency: Urgency): string {
  return `${VET_SEARCH_MAPS_BASE}${encodeURIComponent(buildVetSearchQuery(urgency))}`;
}
```

### `share-payload.ts` — pinned template
```ts
import type { TriageResult } from "@pawcareright/types";
import { strings } from "../strings";
export function buildSharePayload(input: {
  tierLabel: string; result: TriageResult; disclaimerLine: string;
}): string {
  const s = strings.check.result.sections;
  const blocks: string[] = [input.tierLabel, input.result.summary];
  if (input.result.possibleCauses.length)
    blocks.push([s.possibleCauses, ...input.result.possibleCauses.map(c => `- ${c.name}: ${c.whyItFits}`)].join("\n"));
  if (input.result.redFlagsToWatch.length)
    blocks.push([s.redFlagsToWatch, ...input.result.redFlagsToWatch.map(x => `- ${x}`)].join("\n"));
  if (input.result.homeCare.length)
    blocks.push([s.homeCare, ...input.result.homeCare.map(x => `- ${x}`)].join("\n"));
  if (input.result.doNot.length)
    blocks.push([s.doNot, ...input.result.doNot.map(x => `- ${x}`)].join("\n"));
  if (input.result.vetQuestions.length)
    blocks.push([s.vetQuestions, ...input.result.vetQuestions.map(x => `- ${x}`)].join("\n"));
  blocks.push(input.disclaimerLine);
  return blocks.join("\n\n");
}
```
Disclaimer line is ALWAYS the last block (the AC).
