## Interfaces/contracts (executor must match these exactly)

**DTO (`register-device.dto.ts`)**
```
export const EXPO_PUSH_TOKEN_REGEX = /^Expo(nent)?PushToken\[[^\]]+\]$/;
export const SUPPORTED_PLATFORMS = ["ios", "android"] as const;
export type DevicePlatform = (typeof SUPPORTED_PLATFORMS)[number];

class RegisterDeviceDto {
  @ApiProperty({ example: "ExponentPushToken[xxxxxxxxxxxxxxxxxxxxxx]" })
  @IsString() @Matches(EXPO_PUSH_TOKEN_REGEX)  expoPushToken!: string;

  @ApiProperty({ enum: SUPPORTED_PLATFORMS })
  @IsIn(SUPPORTED_PLATFORMS)                    platform!: DevicePlatform;
}
```
(Global `ValidationPipe` is already `whitelist + forbidNonWhitelisted + transform` — no per-DTO config needed. Any failed rule → 400, mapped to `VALIDATION_FAILED` by `AllExceptionsFilter`.)

**Service (`devices.service.ts`)**
```
export interface RegisteredDevice { id: string; expoPushToken: string; platform: string; lastSeenAt: Date; }

@Injectable() DevicesService {
  constructor(private readonly prisma: PrismaService) {}
  register(userId: string, input: { expoPushToken: string; platform: string }): Promise<RegisteredDevice>
}
```
Implementation contract:
- Private `upsertAndPrune(userId, token, platform)` returns the upserted Device by running `this.prisma.$transaction([ upsert, deleteMany ])`:
  - `device.upsert({ where: { expoPushToken: token }, create: { userId, expoPushToken: token, platform }, update: { userId, platform, lastSeenAt: new Date() } })`
  - `device.deleteMany({ where: { userId, platform, expoPushToken: { not: token } } })`
  - Array-form `$transaction` returns `[device, batchPayload]`; take `[0]`.
- `register()` calls `upsertAndPrune`; on `Prisma.PrismaClientKnownRequestError` with `code === "P2002"` (unique-race on the create path) it retries `upsertAndPrune` **exactly once** (the row now exists → update path). Any other error, or a second P2002, propagates.
- Map the Device row → `RegisteredDevice` returning only `{ id, expoPushToken, platform, lastSeenAt }`.
- Import `Prisma` as a **value** (`import { Prisma } from "@prisma/client"`) for the `instanceof` check.

**Controller (`devices.controller.ts`)**
```
@ApiTags("devices") @Controller("devices")
DevicesController {
  @Post() @HttpCode(200)
  register(@CurrentUser() user: { userId: string }, @Body() dto: RegisterDeviceDto): Promise<RegisteredDevice>
}
```
- NOT `@Public()` — the global `JwtAuthGuard` applies; `user` is guaranteed populated. Pass `user.userId` + dto to the service.
- Swagger: `@ApiOkResponse`, `@ApiUnauthorizedResponse`, `@ApiBadRequestResponse`.

**Middleware (`last-seen.middleware.ts`)** — runs BEFORE guards, so `req.user` is NOT available; resolve by header only.
```
export const DEVICE_TOKEN_HEADER = "x-device-token";
@Injectable() LastSeenMiddleware implements NestMiddleware {
  constructor(private readonly prisma: PrismaService) {}
  use(req, res, next) {
    const token = req.header(DEVICE_TOKEN_HEADER);
    if (typeof token === "string" && token.length > 0) {
      void this.prisma.device
        .updateMany({ where: { expoPushToken: token }, data: { lastSeenAt: new Date() } })
        .catch((err) => this.logger.warn(...));  // swallow — never blocks/throws
    }
    next();  // called synchronously, always
  }
}
```
- Use `updateMany` (not `update`) so an unknown token → 0 rows, never a P2025 throw.
- No ownership check (see Risks §C). No throttle (see Risks §B). Nest `Logger`, never `console.*`.
- Types: `import type { Request, Response, NextFunction } from "express"`.

**Module (`devices.module.ts`)**
```
@Module({ imports: [PrismaModule], controllers: [DevicesController], providers: [DevicesService, LastSeenMiddleware] })
DevicesModule implements NestModule {
  configure(consumer: MiddlewareConsumer) { consumer.apply(LastSeenMiddleware).forRoutes("*"); }
}
```

## Risks & the design decisions the planner made (so the checker can scrutinize)
- **A. Reassign-on-conflict (upsert `update` sets `userId`).** A token already owned by another user is moved to the caller. Rationale: an Expo push token is device-bound; on device hand-off / re-login the token must follow the new account or pushes leak to the wrong user. The unique constraint is on `expoPushToken`, so reassignment (not duplicate insert) is the only conflict-free option. Alternative (reject cross-user token with 409) was rejected as worse push hygiene and not requested by the card.
- **B. Middleware is unthrottled fire-and-forget.** Every request bearing `x-device-token` issues one indexed `updateMany` on the unique `expoPushToken` column, off the response path (`void … .catch`; `next()` is synchronous). Per §2-rule-2 minimalism no debounce is added; the card does not ask for one. Acceptable because the write is a single-row update on a unique index and never blocks the response.
- **C. No ownership check in the middleware.** It runs before guards, so `req.user` is unavailable; it updates `WHERE expoPushToken = header` only. The token is itself a high-entropy secret-ish identifier and only the low-value `lastSeenAt` field is touched, so no meaningful abuse surface. Documented for scrutiny.
- **D. No composite `@@index([userId, platform])` / no migration.** The prune query filters `userId` (existing `@@index([userId])`) plus a low-cardinality `platform` residual over a set the prune itself keeps at ~1 row per platform. Adding a composite index for a table with a handful of rows per user is negative-ROI over-engineering (§2-rule-2) and would require a migration, enlarging an "S" task. §6's "index every new query pattern" is satisfied by the existing `userId` index; the upsert/middleware queries ride the `expoPushToken` unique index. Flagged so the checker can overrule if desired.
- **E. Endpoint always returns 200 (via `@HttpCode(200)`), not 201-on-create.** `prisma.upsert` does not report whether it created or updated, so distinguishing 201 vs 200 would need an extra pre-read + its own race. Returning 200 for both create and update keeps the parallel-idempotency AC deterministic (all 5 calls → 200). The card allows "200/201".
- **F. `forRoutes("*")` wildcard binding.** Verified against the installed stack (`@nestjs/platform-express@11.1.28` → `express@5.2.1`, `path-to-regexp@8.4.2`): Nest 11 `RouteInfoPathExtractor.isAWildcard` lists `"*"` as a simple wildcard and `LegacyRouteConverter` auto-converts legacy wildcards, so `"*"` binds globally without a path-to-regexp throw. Fallback if a bootstrap error ever appears: `forRoutes({ path: "{*splat}", method: RequestMethod.ALL })`.
- **G. Expo-token regex `^Expo(nent)?PushToken\[[^\]]+\]$`.** Accepts both the legacy `ExponentPushToken[…]` and current `ExpoPushToken[…]` forms; rejects raw FCM/APNs tokens. Chosen over a looser non-empty check to satisfy the "invalid token shape → 400" AC meaningfully; may reject a future Expo token format (low risk — this is the documented shape).
