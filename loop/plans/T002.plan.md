## Risks & the design decisions the planner made (checker: scrutinize these)
1. **No build step.** Config ships raw `.ts`/`.mjs`/`.json` consumed directly by workspaces (standard for config packages); `packages/config` defines NO `build` task, so `pnpm build` finds nothing to build for it and exits 0. Risk: a future consumer expecting compiled `dist` — mitigated because CLAUDE.md monorepo consumes TS sources via the shared base. Simplest thing that satisfies the ACs.
2. **`moduleResolution: "Bundler"` + `module: "ESNext"` in the base.** Chosen so shared `.ts` sources import cleanly without `.js`-extension gymnastics across web/mobile bundlers; the api workspace (NodeNext) will override these two keys in its own tsconfig when created. Justified as the modern 2026 monorepo default.
3. **`no-console: "error"` with the Nest-Logger exception DEFERRED to the api workspace.** No `api` exists yet, so the base stays strictly `error`; the exception is documented as a comment recipe (api adds an override like `{ files: ["**/main.ts","**/*.logger.ts"], rules: { "no-console": "off" } }`). Simplest correct mechanism; avoids a base override that currently matches nothing.
4. **`eslint-plugin-import-x` over `eslint-plugin-import`** for `import-x/order` — it is the flat-config-native, actively-maintained fork (ESLint 9 ready), avoiding the legacy plugin's partial flat-config support. New dependency → executor must add a one-line justification to `loop/journal.md` at commit time (per CLAUDE.md §2 rule 7), alongside the other five devDeps.
5. **AC verification is transient.** Both `__scratch-*.ts` files are listed in Files ONLY so the scope gate permits them; steps 9–10 require their creation, verification, and deletion. The final tree MUST contain zero `__scratch-*` files — checker should confirm none remain.
6. **No unit test file.** Task adds only config + constant data (no logic), and no test runner is wired yet; adding one would be scope creep. If the checker's policy requires an executable assertion of the constant values, that is a plan-level decision to revisit — flagged here explicitly.
