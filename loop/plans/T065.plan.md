# Plan — T065: Weight chart (mobile, react-native-svg)

## Objective (from card)
Add a mobile weight chart (react-native-svg) for a pet: a line chart of the pet's
weight history with an optional breed typical-adult-range band (from T022 data),
a locale-defaulted kg/lb unit toggle, and an add-weight quick action. All chart
math (scaling, band mapping, unit conversion, ticks, edge domains) is pure and
unit-tested; the component has snapshots for empty / 1-point / many (+ band).

## Safety statement (CLAUDE.md §7 / PRODUCT_SPEC §5)
The typical-range band is **neutral reference information only**. It renders as a
shaded region plus a strictly factual caption ("Typical adult range for {breed}").
There is **no** interpretive, comparative, or judgmental copy anywhere in this task
— no "overweight"/"underweight"/"too heavy", no "possible causes", no "diagnosis"
language, no medication or dosing text. This screen is **not** an AI-result surface,
so it produces no AI output and touches no disclaimer / emergency / triage code;
`<VetDisclaimer/>`, the Emergency interstitial, and the safe-fallback surfaces are
all unaffected. This task does not weaken any Safety Policy surface.

## Key decisions (planner-made — checker, scrutinize these)
1. **Route + entry point.** New flat route `app/weight/[petId].tsx` (mirrors
   `app/check/[category].tsx`, `app/join/[code].tsx`, `app/reminders/edit.tsx`),
   avoiding any restructuring of the existing `app/pets/[id].tsx` file. The pet-home
   "Log weight" quick action is **re-wired** from `/coming-soon` to this route
   (this is the "add-weight quick action" entry). One existing T025 test assertion
   is updated to match (see Files).
2. **Response shape is MIRRORED, not imported.** The mobile hook declares a local
   `WeightSeriesResponse = { points: Array<{ t: string; grams: number }>; sampled: boolean }`
   mirroring `apps/api/.../health-logs.service.ts` — no import across the api boundary
   (same posture the card mandates).
3. **grams are the single source of truth.** Every persisted/transported value is
   integer grams (`weightValueSchema.weightGrams`, int > 0). kg/lb exist only for
   display and input; conversion happens at the edges (pure fns), never in storage.
4. **Unit default = locale, override = persisted toggle.** Imperial regions
   `US`, `LR`, `MM` default to `lb`; everything else `kg` (via `getDeviceRegionCode()`
   from `src/checks/region.ts`). A persisted MMKV+zustand override (mirrors
   `active-pet-store.ts`) wins when set; the in-chart toggle sets it.
5. **Add-weight uses invalidate-refetch, not optimistic** (card allows "simplest
   correct"; the series is downsampled server-side so an optimistic local point
   could not be reconstructed faithfully — invalidate is the honest choice).
6. **react-native-svg is mocked in jest** with lightweight passthrough host
   components (deterministic snapshots, no native/transform dependency). The real
   dependency is added to `package.json` for typecheck + app build (card-mandated;
   see Risks — this grows the dev-client rebuild list).
7. **Band bounds come from `typicalAdultWeightKg` × 1000** → grams; band renders
   only when the pet has a `breedSlug` AND a matching breed row exists in
   `@pawcareright/data`. Otherwise the chart renders normally with no band.

## Files to create/modify (exhaustive — executor may touch NOTHING else)

### Create — pure logic
- `apps/mobile/src/weight/weight-units.ts` — unit domain. Exports:
  `WeightUnit = "kg" | "lb"`; `IMPERIAL_REGIONS = ["US","LR","MM"] as const`;
  `defaultUnitForRegion(regionCode: string | undefined): WeightUnit`;
  `WEIGHT_MIN_GRAMS = 1`, `WEIGHT_MAX_GRAMS = 200000`;
  `gramsToDisplay(grams: number, unit: WeightUnit): number` (kg = g/1000, lb = g/453.592;
  rounded to 1 decimal);
  `formatWeight(grams: number, unit: WeightUnit): string` (e.g. `"25.0 kg"`, unit label appended);
  `parseDisplayToGrams(input: string, unit: WeightUnit): { ok: true; grams: number } | { ok: false; reason: "empty" | "nan" | "range" }`
  (trims, rejects empty/NaN/≤0, converts to grams via round, rejects outside
  `[WEIGHT_MIN_GRAMS, WEIGHT_MAX_GRAMS]`). No React, no imports beyond none.
- `apps/mobile/src/weight/breed-weight-band.ts` — pure band resolver. Exports
  `WeightBand = { minGrams: number; maxGrams: number; breedName: string }` and
  `resolveBreedBand(species: Species, breedSlug: string | null): WeightBand | null`
  using `allBreeds` from `@pawcareright/data` (`find` by `slug` AND `species`;
  returns null when `breedSlug` is null or no row); converts
  `typicalAdultWeightKg.{min,max}` → grams (×1000, rounded).
- `apps/mobile/src/weight/weight-chart-geometry.ts` — pure geometry. Exports
  `GeomPoint = { x: number; y: number }`, `YTick = { value: number; y: number; label: string }`,
  `WeightChartGeometry = { isEmpty: boolean; linePath: string; dots: GeomPoint[]; band: { y: number; height: number } | null; yTicks: YTick[]; yDomain: [number, number]; xDomain: [number, number] }`,
  and `computeWeightChartGeometry(input): WeightChartGeometry` where `input =
  { points: Array<{ t: number; grams: number }>; band: WeightBand | null; unit: WeightUnit; width: number; height: number; padding: number }`.
  Behavior: empty points → `isEmpty:true`, empty path/dots. Single point / flat
  grams-domain → pad the y-domain symmetrically (±1 display-unit worth of grams) to
  avoid div-by-zero and center the value; single point x → chart mid-x. yDomain is
  **expanded to include band.min/max when a band is present** (band-outside-data
  case). Values are mapped to viewBox coords (y inverted: larger grams → smaller y).
  Ticks: 3–5 "nice" ticks across the (display-unit) y-domain, monotonically
  increasing, `label` via `gramsToDisplay`. No React.

### Create — data hook
- `apps/mobile/src/api/health-logs-api.ts` — TanStack Query, mirrors
  `pets-api.ts`/`agenda-api.ts`. Exports `weightSeriesKeys` (`all`, `pet(petId)`),

### Create — UI
- `apps/mobile/src/components/weight-chart.tsx` — presentational. Props
  `{ points: Array<{ t: number; grams: number }>; band: WeightBand | null; unit: WeightUnit; width?: number; height?: number }`.
  Calls `computeWeightChartGeometry`; renders `<Svg testID="weight-chart-svg">` with
  band `<Rect>`, axis `<Line>`s + tick `<Text>`, series `<Path>` + `<Circle>` dots;
  when `geometry.isEmpty` renders `testID="weight-chart-empty"` placeholder (Text
  from strings, NO Svg → no crash). When `band` present renders
  `testID="weight-chart-band-caption"` factual caption. No data fetching inside.
  SVG structural props are exempt from NativeWind (§6); wrapper `View` uses classes.
- `apps/mobile/src/components/add-weight-form.tsx` — the quick-action form. A RN
  `Modal` (or inline card) with a `KeyboardAvoidingView` + `TextInput`
  (`keyboardType="decimal-pad"`, `testID="add-weight-input"`), unit suffix label,
  Save (`testID="add-weight-save"`) and Cancel. On save: `parseDisplayToGrams`;
  on `{ ok:false }` show the mapped inline error (`testID="add-weight-error"`);
  on ok call `onSubmit(grams)`. Props `{ unit; onSubmit; submitting; onClose; visible }`.
  Safe-area insets + keyboard avoidance (§6). No medical/interpretive copy.

### Create — screen

## Tests to write (map to acceptance criteria)
- **AC "unit conversion unit-tested pure"** → `weight-units.test.ts`:
  `gramsToDisplay(25000,"kg")===25.0`; `gramsToDisplay(25000,"lb")≈55.1`;
  `parseDisplayToGrams("25","kg")==={ok:true,grams:25000}`;
  `parseDisplayToGrams("55.1","lb").grams≈25000` (round-trip within tolerance);
  rejects `""`→`empty`, `"abc"`→`nan`, `"0"`/`"-3"`→`range`, `"9999"`kg→`range`
  (>200 kg); `defaultUnitForRegion("US")==="lb"`, `"MM"`/`"LR"`→`"lb"`,
  `"GB"`/`undefined`→`"kg"`.
- **AC "band mapping unit-tested pure"** → `breed-weight-band.test.ts`:
  a real dog slug present in `@pawcareright/data` (e.g. `"labrador-retriever"`)
  with species `DOG` returns `{ minGrams, maxGrams, breedName }` = `typicalAdultWeightKg×1000`;
  `resolveBreedBand("DOG", null)===null`; an unknown slug returns null;
  species mismatch (dog slug queried as `CAT`) returns null.
- **AC "scaling unit-tested pure"** → `weight-chart-geometry.test.ts`:
  empty → `isEmpty:true`, no dots; single point → `isEmpty:false`, one dot at mid-x,
  no NaN in y (flat-domain padded); flat multi-point (all equal grams) → no div-by-zero,
  finite y; many points → dots strictly within `[padding, width-padding]` x and
  `[padding, height-padding]` y, monotonic x for ascending t; **band expands yDomain**
  when band range exceeds data range (band.y/height finite and within plot); yTicks
  count 3–5 and strictly increasing.
- **AC "snapshot for empty / 1-point / many"** → `weight-chart-snapshot.test.tsx`:
  four `toMatchSnapshot()` cases — `points:[]` (asserts `weight-chart-empty`, no
  `weight-chart-svg`); one point; many points; many points **+ band** (asserts
  `weight-chart-band-caption` present). Uses the global react-native-svg mock.
- **Supporting (not ACs, for correctness/§6):**
  `weight-unit-store.test.ts` (override persists, `useWeightUnit` falls back to locale
  default, toggle flips kg↔lb); `health-logs-api.test.ts` (`useAddWeight` posts
  `{ kind:"WEIGHT", value:{ weightGrams } }` and invalidates the series key;
  invalid input is rejected at the form layer, not here); `weight-screen.test.tsx`

## Risks & edge cases (checker: scrutinize)
- **svg-in-jest** (common failure point): handled by mocking `react-native-svg` in
  `jest.setup.ts` (passthrough host components) — the real module never loads under
  jest, so no `transformIgnorePatterns` change is needed. If a snapshot shows raw
  component names that is expected/stable.
- **New native dependency**: `react-native-svg` is card-mandated; it grows the
  founder's **dev-client rebuild** list (native module) — must be journaled + flagged.
  Install must succeed for typecheck/build; if `expo install` cannot resolve offline,
  STOP and write `T065.blocked.md`.
- **Flat / single-point domain**: div-by-zero guarded by symmetric domain padding;
  tested explicitly.
- **Band outside data domain**: yDomain expanded to include band; tested.
- **Breed without a range / pet without breed**: `resolveBreedBand` returns null →
  chart renders with no band, no caption; tested.
- **Unit rounding drift**: grams is the source of truth; display rounds to 1 decimal;
  round-trip (kg/lb → grams → display) tested within tolerance to bound drift.
- **Route re-wire**: changing the log-weight target updates exactly one existing
  assertion in `pet-home-screen.test.tsx`; the pet-home **snapshot** is unaffected
  (onPress handlers are not serialized).
- **Decisions 1 and 5** (flat route + entry re-wire; invalidate-not-optimistic) are
  the only non-forced design choices — flagged here for review.
