# Plan — T054: Care template packs

## Objective (from card)
Ship `packages/data/care-templates`: DOG/CAT × life-stage × region-protocol-group care schedules (core vaccines, deworming, flea/tick, dental, grooming), encoded as our own structured guidance (source names in comments only, no copied prose), each item carrying an rrule from our validated subset and a schema-enforced "confirm with your vet" note. Deps: T005 only (pure data package — no DB, no API, no UI).

## Safety statement (this is a PRODUCT_SPEC §5-ADJACENT surface — HIGH scrutiny)
Care schedules are veterinary-medicine content. This plan does NOT weaken any §5 surface; it feeds F4 (Care Plans & Reminders). Enforced invariants:
- **Confirm-with-your-vet is schema-enforced, not convention.** Every item's `note` MUST contain the pinned `VET_CONFIRM_SENTENCE`; `careTemplateItemSchema` carries a `.refine` that REJECTS any note lacking it (Decision R4). A note-without-it is structurally impossible to ship.
- **No dosing, ever (§5 rule 4 / §7 rule 2).** Titles/notes are qualitative and category-level ("flea/tick preventive treatment", not a brand or a dose). A mechanical scan test enforces: no numeric-unit dosing (`\b\d+\s*(mg|ml|mcg|g|kg|iu)\b`), no `mg/kg`, no `/diagnos/i`, and no product-brand names (denylist) across every title+note (Decision R5).
- **No drug/product brand names as recommendations.** Vaccines named by antigen/disease in plain language (e.g. "rabies vaccination", "core vaccination series"), never a manufacturer/brand. Brand denylist scan test (Decision R5).
- **Sources named in comments only** (WSAVA, AAHA, AAFP, WSAVA/WHO rabies data, ESCCAP/TROCCAP for parasites) — no copied text, no URLs; knowledge re-expressed in our own words (Decision R9).
- **Fail-safe resolution.** Unknown region group → `DEFAULT` group. Unknown/absent life-stage → `ADULT` baseline (Decision R6 — this is the ONE non-obvious safety call; see Risks: auto-scheduling an intensive juvenile vaccine series onto an unknown-age animal risks harmful over-vaccination, so §5 "animal safety beats every goal" overrides a naive "more reminders = safer"; the adult pack still directs the owner to confirm the true life stage with their vet).
This task does not touch disclaimers, the emergency interstitial, or dosing prohibitions. No SAFETY-ESCALATION.

## Key decisions (design shape)
1. **Data-model shape — base layer + regional vaccine overlay, merged by the resolver (avoids 54 copies) (R1).** A region-agnostic **base** `Record<Species, Record<LifeStage, item[]>>` carries the bulk (deworming, flea/tick, dental, grooming, species-universal core DHPP/FVRCP series). A **vaccine overlay** carries only the region-varying vaccines (rabies +/- leptospirosis). `resolveCareTemplate` = `base[species][stage] ∪ overlay-for-group[species][stage]`. Overlay is defined for `DEFAULT` plus only the groups that differ; a group with no explicit overlay entry uses `DEFAULT`. The full-matrix meta-test **materializes all 54 resolved cells** and validates each, so auditability is preserved on the *resolved product* without 54 near-duplicate source blocks. (Card: "base schedules + regional overrides/extensions".)
2. **Protocol-group mapping lives in this package, distinct from T049 hotline regions (R2).** `PROTOCOL_GROUPS = NA/EU/UK/IN/BR/MENA/SEA/AU/DEFAULT` (coarse vaccine-protocol grouping) — a DIFFERENT, coarser axis than T049's ISO-alpha-2 hotline regions. `protocolGroupForCountry(iso2?)` maps ISO-3166 alpha-2 → group via a small explicit map; unknown/undefined → `DEFAULT`.
3. **Life-stage boundaries (R3), source-backed (AAHA canine / AAHA-AAFP feline life-stage guidelines), collapsed to 3 stages.** `LIFE_STAGES = PUPPY_KITTEN / ADULT / SENIOR`. DOG: PUPPY_KITTEN 0–11mo, ADULT 12–83mo, SENIOR ≥84mo (7y). CAT: PUPPY_KITTEN 0–11mo, ADULT 12–119mo, SENIOR ≥120mo (10y). Giant-breed dogs age faster — noted in copy as a vet-confirm point, not modeled per-breed.
4. **Anchor semantics — only two anchors, both resolvable from the existing Pet model (R7).** `PET_AGE` (offset measured from the pet's date of birth; `startOffsetDays` = the age in days when the item first applies, e.g. 42 = 6 weeks) and `PLAN_START` (offset from the day the owner instantiates the plan). Puppy/kitten core-series + juvenile rabies use `PET_AGE`; adult/senior boosters + all deworming/flea-tick/dental/grooming use `PLAN_START`. **No `PET_ADOPTION` anchor** — adoption date is not on `Pet` (pet.ts has only `birthDate`/`ageEstimateMonths`); inventing it would be unresolvable by T055. T055 owns turning `PET_AGE`+`startOffsetDays` into a real DTSTART against a pet's `birthDate`/`ageEstimateMonths` and deciding behavior when age is unknown.

## Interfaces/contracts (executor must match)
```ts
// packages/data/src/care-templates/schema.ts
export const TEMPLATE_CATEGORIES = ["vaccine", "deworming", "flea-tick", "dental", "grooming"] as const;
export const TEMPLATE_ANCHORS = ["PET_AGE", "PLAN_START"] as const;
export const PROTOCOL_GROUPS = ["NA", "EU", "UK", "IN", "BR", "MENA", "SEA", "AU", "DEFAULT"] as const;
export const LIFE_STAGES = ["PUPPY_KITTEN", "ADULT", "SENIOR"] as const;

/** Pinned user-facing sentence every item note MUST include (Decision R4). */
export const VET_CONFIRM_SENTENCE =
  "Confirm the right timing and products for your pet with your veterinarian.";

export const careTemplateItemSchema = z.strictObject({
  id: slugSchema,                                  // unique within a resolved pack (asserted by meta-test)
  category: z.enum(TEMPLATE_CATEGORIES),
  title: z.string().min(1).max(120),               // plain language, no brand names, no dosing
  note: z.string().min(1).max(600)
        .refine((s) => s.includes(VET_CONFIRM_SENTENCE), { message: "note must include VET_CONFIRM_SENTENCE" }),
  rrule: rruleSchema,                              // z.string().refine(isValidRRule) from @pawcareright/types
  anchor: z.enum(TEMPLATE_ANCHORS),
  startOffsetDays: z.number().int().min(0),
  emphasis: z.boolean().default(false),           // true = region strongly emphasizes (e.g. rabies in IN)
});
export type CareTemplateItem = z.infer<typeof careTemplateItemSchema>;       // resolved (emphasis defaulted)
export type CareTemplateItemInput = z.input<typeof careTemplateItemSchema>;  // authoring shape (emphasis optional)

export const CATEGORY_TO_REMINDER_TYPE: Record<TemplateCategory, ReminderType> = {
  vaccine: "VACCINE", deworming: "PARASITE", "flea-tick": "PARASITE", dental: "DENTAL", grooming: "GROOMING",
};

// index.ts — resolver API for T055 (instantiate)/T059 (wizard)
export interface ResolvedCareTemplateItem extends CareTemplateItem { reminderType: ReminderType; }
export interface ResolvedCareTemplate {
  species: Species; lifeStage: LifeStage; group: ProtocolGroup;  // group = the group actually used (DEFAULT if fell back)
  items: ResolvedCareTemplateItem[];                              // non-empty; ids unique within pack
}
export function resolveCareTemplate(species: Species, lifeStage: LifeStage, group: ProtocolGroup): ResolvedCareTemplate;
export function resolveCareTemplateForPet(input: {
  species: Species; ageMonths?: number | null; countryCode?: string | null;
}): ResolvedCareTemplate; // lifeStage via resolveLifeStage (absent→ADULT); group via protocolGroupForCountry (unknown→DEFAULT)

// life-stages.ts
export function lifeStageForAgeMonths(species: Species, ageMonths: number): LifeStage;
export function resolveLifeStage(species: Species, ageMonths?: number | null): LifeStage; // null/undefined → "ADULT"
// protocol-groups.ts
export function protocolGroupForCountry(countryCode?: string | null): ProtocolGroup;      // unknown/undefined → "DEFAULT"
```

## Content spec (blueprint — executor authors following the pattern; only the AC cases are pinned in tests)

**Life-stage boundaries (R3):**
| species | PUPPY_KITTEN | ADULT | SENIOR | source (comment) |
|---|---|---|---|---|
| DOG | 0–11 mo | 12–83 mo | ≥84 mo | AAHA canine life-stage |
| CAT | 0–11 mo | 12–119 mo | ≥120 mo | AAHA/AAFP feline life-stage |

**Note pattern (every item):** `<one or two plain-language guidance sentences, category-level, no doses/brands> ` + `VET_CONFIRM_SENTENCE`. Example (rabies, IN): "Rabies is endemic across much of India and rabies vaccination is strongly emphasized for dogs and cats here, and may be legally expected. " + `VET_CONFIRM_SENTENCE`.

**BASE_SCHEDULES item blueprint (region-agnostic; ids stable, unique within a resolved pack):**
| id | category | stage(s) | anchor / offset | rrule (subset) | intent |
|---|---|---|---|---|---|
| `core-vaccine-series` | vaccine | PUPPY_KITTEN | PET_AGE / 42 | `RRULE:FREQ=WEEKLY;INTERVAL=3;COUNT=3` | DHPP (dog)/FVRCP (cat) primary series, ~3 doses 3wk apart from ~6wk |
| `core-vaccine-booster` | vaccine | ADULT, SENIOR | PLAN_START / 0 | `RRULE:FREQ=YEARLY;INTERVAL=3` | core booster (triennial; note: some cores annual — confirm) |
| `deworming-juvenile` | deworming | PUPPY_KITTEN | PET_AGE / 14 | `RRULE:FREQ=WEEKLY;INTERVAL=2;COUNT=6` | intensive juvenile deworming (ESCCAP-style cadence) |
| `deworming-routine` | deworming | ADULT, SENIOR | PLAN_START / 0 | `RRULE:FREQ=MONTHLY;INTERVAL=3` | quarterly-ish adult deworming |
| `flea-tick-preventive` | flea-tick | all 3 | PLAN_START / 0 | `RRULE:FREQ=MONTHLY` | monthly preventive; NOTE covers regional/seasonal variance (card: "monthly-with-regional-notes") |
| `dental-check` | dental | all 3 | PLAN_START / 0 | `RRULE:FREQ=YEARLY` | annual dental review (senior note: may be more frequent) |
| `grooming-routine` | grooming | all 3 | PLAN_START / 0 | `RRULE:FREQ=MONTHLY` | routine grooming/coat/nail check |

**VACCINE_OVERLAYS blueprint (region-varying vaccines only; the DEFAULT entry is mandatory so every pack has a rabies item):**
- `DEFAULT`: `rabies-core` (vaccine). PUPPY_KITTEN: PET_AGE / 98 (~14wk), `RRULE:FREQ=YEARLY` first-year then per law; ADULT/SENIOR: PLAN_START / 0, `RRULE:FREQ=YEARLY`. Note: "rabies requirements and importance vary by country and local law". `emphasis: false`.
- `IN` (MANDATORY — the AC): `rabies-emphasis-in` (vaccine), `emphasis: true`, note emphasizing endemic rabies + strong recommendation, for PUPPY_KITTEN + ADULT + SENIOR (same anchors/rrules as DEFAULT rabies).
- Other groups where the executor can responsibly encode a delta (source: WSAVA/WHO rabies endemicity, ESCCAP/TROCCAP): rabies-endemic regions (`BR`, `MENA`, `SEA`) → rabies `emphasis: true`; `AU` → rabies-free region, replace routine rabies with a travel-only note item (or omit rabies) — AU packs still non-empty via base; `NA`/`EU`/`UK` → keep DEFAULT-style rabies (`emphasis: false`) and MAY add a `leptospirosis-regional` item (lifestyle/regional risk note). Groups left undefined inherit `DEFAULT`. Regional emphasis assignments are content the C-checkpoint vet review confirms (R8) — tests pin ONLY the IN + DEFAULT rabies cases (below) to avoid brittle medicine assertions.

**§7 / brand rules:** titles/notes carry NO manufacturer/brand names. Denylist for the scan test (non-exhaustive, extendable): `frontline, bravecto, nexgard, simparica, seresto, advantage, advantix, revolution, heartgard, milbemax, drontal, panacur, apoquel, comfortis, credelio` (case-insensitive, word-boundary).

## Resolver API (for T055 instantiate / T059 wizard / T057 collapse)
- `resolveCareTemplate(species, lifeStage, group)` → `ResolvedCareTemplate` with non-empty `items`, each carrying its derived `reminderType`. Group resolution: use `VACCINE_OVERLAYS[group]` if defined, else `VACCINE_OVERLAYS.DEFAULT`; returned `group` reflects the group actually applied (the requested group if it had an explicit overlay OR the base-only dimensions still use the requested group label; DEFAULT only when the requested group is not in `PROTOCOL_GROUPS`). Base items are group-independent.
- `resolveCareTemplateForPet({species, ageMonths?, countryCode?})` → computes `lifeStage = resolveLifeStage(species, ageMonths)` (absent → ADULT) and `group = protocolGroupForCountry(countryCode)` (unknown → DEFAULT), then delegates. This is the single honest entry point T059 calls; T055 maps each `ResolvedCareTemplateItem` (`anchor`+`startOffsetDays`+`rrule`+`reminderType`+`title`+`note`) onto a real `Reminder` for a pet.
- Resolver NEVER throws for a valid `Species`/`LifeStage`; an out-of-enum `group` string is narrowed to `DEFAULT`.

## Risks & design decisions (numbered — for checker scrutiny)
- **R1 — Base + regional-overlay merge, not 54 flat cells.** DRY authoring (base bulk written once; overlay carries only region-varying vaccines; undefined groups inherit `DEFAULT`). Auditability preserved because the 54-cell meta-test materializes and validates every *resolved* pack. Diverges from T035's flat-rows choice, justified: toxins have no shared structure across rows, whereas care schedules are ~90% region-identical, so flat copies would invite silent regional drift (§2 rule 2).
- **R2 — Protocol groups are a NEW coarse axis, local to this package**, deliberately separate from T049's ISO hotline regions. `protocolGroupForCountry` (ISO2→group, unknown→DEFAULT) is the mapping layer, living beside the data it serves.
- **R3 — Life-stage boundaries** (DOG senior ≥84mo, CAT senior ≥120mo) collapse the AAHA/AAFP finer stages into 3; sourced in comments. Boundary-inclusive at the lower edge (12mo = ADULT). Giant-breed earlier aging handled via note, not per-breed logic.
- **R4 — Confirm-with-vet enforced by a schema `.refine` on `note`** (substring of the pinned `VET_CONFIRM_SENTENCE`), exported for reuse by T059 snapshot tests. Convention alone was explicitly rejected by the card.
- **R5 — §7 scan targets `title`+`note` only** (not `id`, which is a slug). Bans dosing numeric-units, `mg/kg`, `/diagnos/i`, and a brand-name denylist. Vaccines/parasiticides are named by disease/category, never by brand or dose.
- **R6 — Unknown/absent life-stage → `ADULT`, NOT the juvenile pack.** This is the one place I depart from a literal "fail toward more care" reading: scheduling an intensive multi-dose juvenile vaccine series onto an unknown-age (possibly adult) animal risks harmful over-vaccination, so §5 supremacy ("animal safety beats every goal") wins over "more reminders." The adult baseline is universally applicable, harmless, and still directs the owner (via every note) to confirm the true life stage with a vet. Unknown *group* → `DEFAULT` (a safe global baseline). Flagged prominently for checker/vet scrutiny.
- **R7 — Only `PET_AGE` and `PLAN_START` anchors** — both resolvable from the existing `Pet` model (`birthDate`/`ageEstimateMonths` + today). No `PET_ADOPTION` (no adoption date on `Pet`). Turning anchors into concrete dates (and handling unknown age) is T055's job.
- **R8 — Regional vaccine emphasis assignments are content, pinned in tests only for IN (the AC) + DEFAULT (contrast).** Other groups' emphasis/lepto deltas follow the blueprint (WSAVA/WHO endemicity) but are left to the C-checkpoint veterinarian review; tests deliberately avoid asserting per-region medicine to prevent brittle/wrong assertions. YEARLY rrules use no `BYMONTH`/`BYMONTHDAY`, so no template can hard-code a Feb-29 anchor (T053 carry-forward): Feb-29 handling is inherited from T055's next-fire logic, not encoded here.
- **R9 — Authored as TypeScript files with source-name header comments** (WSAVA, AAHA, AAFP, ESCCAP/TROCCAP, WHO rabies) — names only, no copied prose, own words (mirrors T035 R9). Split base vs overlay for reviewability and 2-pass execution resilience.
