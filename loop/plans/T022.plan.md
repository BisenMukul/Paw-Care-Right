## Interfaces / contracts (executor must match exactly)

### Breed schema — `packages/data/src/breeds/schema.ts`
```
SIZE_CLASSES = ["TOY","SMALL","MEDIUM","LARGE","GIANT","UNKNOWN"] as const
sizeClassSchema = z.enum(SIZE_CLASSES)

BREED_SPECIES = ["DOG","CAT"] as const           // kept local to data (leaf pkg; no data→types dep)
breedSpeciesSchema = z.enum(BREED_SPECIES)

slugSchema = z.string().regex(/^[a-z0-9]+(?:-[a-z0-9]+)*$/)   // kebab-case

// Rows AS STORED in the JSON files (species is IMPLIED by the file, not repeated per row)
breedRowSchema = z.object({
  slug: slugSchema,
  name: z.string().min(1),
  sizeClass: sizeClassSchema,
  typicalAdultWeightKg: z.object({
    min: z.number().positive(),
    max: z.number().positive(),
  }).refine(w => w.max >= w.min, { message: "max must be >= min" }),
})

// Public, fully-resolved breed (species attached by the loader)
breedSchema = breedRowSchema.extend({ species: breedSpeciesSchema })
type Breed = z.infer<typeof breedSchema>
```

### Loader — `packages/data/src/breeds/index.ts`
```
import dogsRaw from "./dogs.json"
import catsRaw from "./cats.json"
// parse each row through breedRowSchema, attach species, re-validate via breedSchema
export const dogBreeds: readonly Breed[] = Object.freeze(dogsRaw.map(r => breedSchema.parse({ ...r, species: "DOG" })))
export const catBreeds: readonly Breed[] = Object.freeze(catsRaw.map(r => breedSchema.parse({ ...r, species: "CAT" })))
export const allBreeds: readonly Breed[] = Object.freeze([...dogBreeds, ...catBreeds])
```
Parsing at module load is the runtime validation layer; `dataset.spec.ts` is the build/test-time layer.

### Matcher — `packages/data/src/breeds/search.ts`
```
normalize(s): string
  = s.normalize("NFD").replace(/\p{Diacritic}/gu, "")   // strip accents
      .toLowerCase().replace(/[^a-z0-9]+/g, " ").trim().replace(/\s+/g, " ")

levenshtein(a, b): number   // standard DP, hand-rolled, no dependency

searchBreeds(species: "DOG" | "CAT", q: string, limit = 20): Breed[]
```
Algorithm:
1. `pool` = dogBreeds or catBreeds for `species`.
2. `nq = normalize(q)`. If `nq === ""` → return the FULL pool sorted by `name` (A→Z) (no cap; it is a picker list).
3. For each breed compute `tokens = normalize(name).split(" ")`, `initials = tokens.map(t => t[0]).join("")`, `norm = normalize(name)`. Assign the BEST (lowest) matching tier:
   - Tier 0 — exact: `norm === nq`.
   - Tier 1 — prefix: `norm.startsWith(nq)` OR any `token.startsWith(nq)`.
   - Tier 2 — initials: `initials === nq` OR `initials.startsWith(nq)`.
   - Tier 3 — substring: `norm.includes(nq)`.
   - Tier 4 — fuzzy: `min(levenshtein(nq, norm), min over tokens levenshtein(nq, token)) <= threshold`, where `threshold = nq.length <= 3 ? 1 : 2`.
   - No tier matched → exclude.
4. Sort by `(tier asc, name.length asc, name A→Z)`; return first `limit` (20).

Ranking is exactly prefix > initials > substring > levenshtein (Tier 0 exact folds above prefix). The `gsd` case: `"German Shepherd Dog"` → tokens `[german,shepherd,dog]` → initials `"gsd"` → Tier 2 exact-initials → ranks above any incidental substring/fuzzy → returned first.

### Service — `apps/api/src/breeds/breeds.service.ts`
- Injects `RedisService`. Return type `Breed[]` (imported from `@pawcareright/data`).
- `search(species, q)`:
  - `nq = normalize(q ?? "")`; `memKey = ` `${species}:${nq}` ; `redisKey = ` `pawcareright:breeds:${species}:${nq === "" ? "_all" : nq.replace(/\s+/g, "_")}`.
  - In-memory L1: private `Map<string, Breed[]>` on the (singleton) service. On hit → return.
  - Redis L2: `try { const cached = await redis.get(redisKey); if (cached) { parse JSON, set L1, return } } catch { Logger.warn(...) }` — never let a Redis failure break the endpoint (graceful degradation, ARCHITECTURE §8).
  - Compute: `results = searchBreeds(species, q ?? "")`. `try { await redis.set(redisKey, JSON.stringify(results), 3600) } catch { Logger.warn(...) }`. Set L1 (evict oldest entry when `map.size > 1000` via `map.delete(map.keys().next().value)`). Return.
- Use Nest `Logger` (never `console.*`). TTL = `3600` s.

### Controller — `apps/api/src/breeds/breeds.controller.ts`

### Dataset authoring guidance (the bulk of the work)
- Author from general knowledge of AKC/FCI (dogs) and TICA/CFA (cats) breed lists to reach ~340 dog rows and ~70 cat rows. Approximate typical adult weight ranges in **kg** are acceptable — plausibility, not precision, is required (these are not a safety/dosing surface).

## Risks & the design decisions the planner made (scrutinize these)
1. **Throttling: `@SkipThrottle()` on the breeds controller.** The card specifies no rate class for breeds; the global 100/60s default would 429 the p95 test's 100 same-IP requests. Breeds is public, cheap, fully-cached static data, so skipping the limiter is defensible. Alternative rejected: a `THROTTLE_BREEDS` override at a high limit (more moving parts, still arbitrary). If abuse protection is later desired, that is a fix() micro-task.
2. **Weight unit = kg (float), not grams.** `Pet.weightGrams` is integer grams, but breed reference ranges are conventionally kg with decimals and must stay human-authorable/reviewable in JSON. The T065 band consumer converts grams→kg. This unit mismatch is intentional and flagged.
3. **Breed Zod schema lives in `packages/data`, not `packages/types`.** Data is the single producer+validator of the dataset (and the future web-SEO SSG source per ARCHITECTURE §2), so the schema and pure `searchBreeds` live there as a leaf package. Cost: species enum is duplicated a third time (Prisma, `packages/types`, here) — accepted to keep `packages/data` dependency-free.
4. **Redis L2 cache has questionable ROI** for an already-in-memory dataset (search is ~1–3ms over ~340 rows), but the card mandates both layers, so it is implemented as a best-effort second-level cache (TTL 3600s) wrapped in try/catch so a Redis outage degrades gracefully rather than failing the endpoint.
5. **p95 test flakiness under CI variance.** Mitigated by 5 warm-up requests (cache-hit path), n=100 samples, and the in-memory fast path. If CI proves flaky, loosening n / threshold is a follow-up fix(test) — not a code defect.
6. **api resolves `@pawcareright/data` two ways** (jest→src via moduleNameMapper like `config`; tsc→dist d.ts via tsconfig path like `types`). This mirrors existing precedents but is a mixed convention; the dist path requires turbo `^build` to have built `packages/data` before api typecheck/build (already guaranteed by `turbo.json`).
7. **Fuzzy threshold (`<=1` for q≤3 chars, else `<=2`)** is a judgement call to balance typo tolerance vs. over-matching; asserted via the typo test but may need tuning if it surfaces false positives.
8. **Empty `q` returns the entire species list (~340 items), uncapped.** Chosen so the endpoint doubles as the breed-picker source; flagged as a payload-size decision (acceptable for a static list; response is cached).

## Amendment A0 (orchestrator, pre-execution) — throttling stays ON in production
The plan's `@SkipThrottle()` on `GET /v1/breeds` is REVERSED: a public, unauthenticated endpoint must keep rate limiting (T017 posture; the global 100/min default applies — no custom class needed, none mandated by the card). The reason SkipThrottle was reached for — the p95 e2e fires ~100+ rapid requests that would trip the limit — is solved TEST-SIDE: the p95 spec's TestingModule overrides the ThrottlerGuard APP_GUARD provider with a pass-through (`{ canActivate: () => true }`) in THAT spec only (mirrors the e2e overrideProvider pattern). All other breeds e2e cases run under the real throttler (they are few). Production route carries NO throttle decorator.
