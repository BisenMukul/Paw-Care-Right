# Plan — T035: Food & toxin dataset + lookup service

## Objective (from card)
Ship a curated ≥220-item dog/cat food-&-toxin dataset with per-species verdicts in `packages/data/toxins`, a normalizer (plurals/misspellings/synonyms), and a provider-injected lookup service: dataset hit → verdict; miss → caution-biased AI fallback → cache write. Deps: T005 only (no DB tasks).

## Safety statement (this is a PRODUCT_SPEC §5 surface — HIGH scrutiny)
Toxin verdicts are safety-critical content: a wrong `safe` can kill a pet. This plan does NOT weaken any §5 surface — it hardens the F3 food-safety module. Enforced invariants:
- **Fail upward everywhere.** When authoritative sources disagree or nuance is unestablished, the executor picks the **more severe** verdict (tie-break rule, §5 rule 2 / CLAUDE §7 rule 5).
- **AI never returns `safe`.** The fallback service floors any AI `safe` verdict to `caution` (Decision R5). A hallucinated `safe` is the worst failure mode and is structurally impossible to surface.
- **Uncertainty → conservative fallback, never a silent guess.** Schema-invalid AI output or a provider error yields a frozen `caution` fallback answer that is NOT cached (Decisions R6).
- **No dosing, ever (§5 rule 4 / §7 rule 2).** Notes encode nuance **qualitatively** (e.g. "dark and baking chocolate are more dangerous than milk chocolate") with **no mg/ml/mg-per-kg numbers**. A mechanical scan test enforces this over `note` + `quantityNuance` (Decision R8).
- Drug **names** as toxin entries (e.g. `ibuprofen`, verdict `emergency`) are **warnings, not recommendations** — permitted by design and explicitly distinguished from dosing advice (Decision R8).
This task does not touch disclaimers, the emergency interstitial, or dosing prohibitions; it feeds them. No SAFETY-ESCALATION.

```ts
// packages/types/src/food-safety.ts
export const FOOD_VERDICTS = ["safe", "caution", "toxic", "emergency"] as const;
export const foodVerdictSchema = z.enum(FOOD_VERDICTS);
export type FoodVerdict = z.infer<typeof foodVerdictSchema>;
export const FOOD_VERDICT_SEVERITY: Record<FoodVerdict, number> =
  { safe: 0, caution: 1, toxic: 2, emergency: 3 };
export const foodSafetyAnswerSchema = z.strictObject({
  verdict: foodVerdictSchema,
  note: z.string().min(1).max(600),
}).superRefine(assertNoDosingOrDiagnosis); // §7: /diagnos/i, /\b\d+\s*(mg|ml|mcg|g|kg|iu)\b/i, /mg\s*\/\s*kg/i, /per\s+(kg|pound|lb)\b/i
export type FoodSafetyAnswer = z.infer<typeof foodSafetyAnswerSchema>;
export function parseFoodSafetyAnswer(raw: unknown):
  | { ok: true; result: FoodSafetyAnswer } | { ok: false; reason: string };
export const FOOD_SAFETY_FALLBACK: Readonly<FoodSafetyAnswer>; // { verdict: "caution", note: "...avoid giving it ... contact a licensed veterinarian." } (deep-frozen, self-validates)
```

## Dataset spec

**Categories + minimum counts** (sum of minimums = 200; aggregate assertion **≥220** → ~20 authoring slack, aim ≥235):
| Category | id value | min |
|---|---|---|
| human foods (fruit/veg/nut/sweetener/drink/dairy) | `human-food` | 85 |
| plants (houseplants, garden, flowers) | `plant` | 50 |
| household chemicals | `household-chemical` | 22 |
| human meds (as toxin entries) | `human-med` | 22 |
| pest baits (rodenticide/insecticide/molluscicide) | `pest-bait` | 12 |
| other (essential oils, recreational substances) | `other` | 9 |

**Per item:** kebab `id`, canonical `name`, `category`, `verdicts:{dog,cat}`, qualitative `note`, optional `quantityNuance`, `aliases[]` (synonyms + common misspellings — these feed the normalizer).

**Five pinned anchor cases** (transcribe EXACTLY; tests assert these per-species verdicts):
| id | dog | cat | required aliases (non-exhaustive) | nuance |
|---|---|---|---|---|
| `grapes` | `emergency` | `toxic` | grape, raisin, raisins, sultana, currant | any amount is dangerous for dogs |
| `xylitol` | `emergency` | `caution` | birch sugar, sugar free gum, sugar-free gum, sweetener | dog toxicity is severe/rapid; cat effect not well established |
| `true-lilies` | `caution` | `emergency` | lily, lilies, easter lily, tiger lily, daylily, stargazer, lilium | true (Lilium/Hemerocallis) lilies; cats — even pollen/water |
| `onion` | `toxic` | `toxic` | onions, garlic, chives, leek, shallot, allium, onion powder, garlic powder | all allium forms incl. powders; cats more sensitive |
| `chocolate` | `toxic` | `toxic` | choco, cocoa, cacao, dark chocolate, milk chocolate, baking chocolate, chocolot, chocalate | dark & baking chocolate more dangerous than milk; large amounts an emergency |

**Sourcing/citation rule (R9):** each category file carries a header comment naming the authoritative references used (e.g. `ASPCA Animal Poison Control Center`, `Pet Poison Helpline`, `Merck Veterinary Manual`). **Names only — no copied prose, no URLs.** Knowledge is encoded in our own words.

**Fail-upward tie-break:** where sources disagree or a species' toxicity is unestablished, choose the **more severe** verdict.

## Normalizer semantics (R10)
- `normalizeItem`: NFKD → strip combining diacritics → lowercase → drop apostrophes → `[^a-z0-9]+`→single space → trim (mirrors `packages/data/src/breeds/search.ts` `normalize`).
- **Singularization:** per-token trailing-plural strip (`grapes`→`grape`, `onions`→`onion`, `lilies`→`lily`, `berries`→`berry`), applied to query and index keys.
- **Misspellings:** primarily data-driven via each entry's `aliases[]` (e.g. `chocolot`, `chocalate` listed on `chocolate`); a small curated map in `normalize.ts` covers the highest-frequency ones. Hand-rolled — **no NLP/stemming dependency** (CLAUDE §2 rule 7).
- **Index:** built once from `toxins` — `Map<normalizedAlias, id>` over `normalizeItem(name)` + every `normalizeItem(alias)` (+ singularized forms). A miss returns `undefined` (feeds AI fallback). Multi-word aliases supported (`sugar free gum`→`xylitol`).
- `findToxinEntry` returns the entry; `toxinVerdictFor` maps to the species verdict.

## Fallback service semantics
Flow of `checkFoodSafety(species, rawItem, deps)`:
1. `entry = findToxinEntry(rawItem)`. **Hit** → return `{source:"DATASET", verdict: entry.verdicts[species.toLowerCase()], note, quantityNuance, cached:false}`. **Dataset hits bypass the cache entirely** (Decision R7 — cache is AI-only).
2. Miss → `key = foodCacheKey(species, normalizeItem(rawItem))`; `cached = await deps.cache.get(key)`. Present → return `{source:"AI", ...cached, cached:true}`.
3. Absent → `deps.provider.generate(buildFoodSafetyPrompt(...))` → `extractJsonCandidate` (reuse from `../triage/extract-json`) → `parseFoodSafetyAnswer`:
   - **ok** → **caution floor:** if `verdict === "safe"` coerce to `"caution"` (Decision R5); `await deps.cache.set(key, answer)`; return `{source:"AI", ...answer, cached:false}`.
   - **parse/schema fail** → return `{source:"FALLBACK", ...FOOD_SAFETY_FALLBACK, cached:false}` — **NOT cached** (Decision R6).
   - **provider throws** → caught → same `FALLBACK`, **not cached**. Service NEVER throws.

**Cache port contract:** key = `species:normalizedItem` (ARCHITECTURE line 63). `CachedAnswer` stores only `{verdict, note}`. The Prisma-backed impl (hitCount, answerJson column) is the future api food module's job; T035 ships the port + `InMemoryAnswerCache` fake only.

## Risks & design decisions (numbered — for checker scrutiny)
- **R1 — Scope split.** Dataset+normalizer in `packages/data`; orchestration in `packages/ai` behind `AnswerCachePort`; NO Prisma/endpoint. Justified: deps are T005-only, no `AnswerCache` model exists, no endpoint card exists; the accept criterion "fallback path caches (test with fake provider)" is fully satisfiable with an injected in-memory port.
- **R2 — Verdict enum + AI answer schema live in `packages/types`** (not `packages/data`); `packages/data` gains a `@pawcareright/types` workspace dep. Justified: CLAUDE §6 "single source of truth" for a genuinely cross-package shape (data/ai/api/mobile/web). This **diverges** from the breeds precedent (which kept `BREED_SPECIES` local to preserve `data`'s leaf status) — but that was a *duplicate* of an existing `Species` enum, whereas `FoodVerdict` is new and has no prior home; duplicating it would violate §6's "never hand-write a duplicate." No cycle (`types` depends on nothing).
- **R3 — `packages/ai` gains a `@pawcareright/data` workspace dep.** Internal only (no external package → CLAUDE §2 rule 7 not triggered). No cycle: `data` does not import `ai`. `pnpm-lock.yaml` regenerates.
- **R4 — Pinned anchor verdicts apply the fail-upward tie-break** (cat grapes=`toxic`, cat xylitol=`caution`, dog/cat chocolate=`toxic` with emergency nuance, true-lily cat=`emergency`). If a reviewing veterinarian prefers different baselines these five rows + tests are the single place to adjust.
- **R5 — Caution floor:** AI `safe` is coerced to `caution` before return/caching. A model must never be able to surface `safe` for an unknown item.
- **R6 — Not cached on failure:** schema-invalid AI output and provider errors return the frozen `caution` fallback and are NOT written to the cache (so a transient failure can't poison future lookups).
- **R7 — Dataset hits bypass the cache** (the cache exists only to amortize AI calls; curated verdicts are already authoritative and instant).
- **R8 — §7 dosing scan targets `note` + `quantityNuance` only**, never `name`/`aliases`/`id`. Drug **names** as toxin entries are warnings (permitted); dosing **numbers/recommendations** are banned. The scan matches numeric-unit and mg/kg patterns + `/diagnos/i`, not the presence of a drug name.
- **R9 — Dataset authored as TypeScript category files** (not JSON) so source citations can live in header comments per the card ("cite source names in dataset comments"). Split by category for reviewability and so the dataset lands verifiable before the service (supports a possible 2-pass execution).
- **R10 — Normalizer is hand-rolled** (case/diacritics/whitespace + per-token singularization + a small misspelling map), with synonyms/misspellings primarily carried in each entry's `aliases[]`. No NLP/stemming dependency.
