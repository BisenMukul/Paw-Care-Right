6. Run the self-verify commands (see below) and confirm `pnpm i` completes cleanly and `pnpm turbo run build --dry` succeeds and prints the (currently empty) workspace/task list.

## Risks & the ONE decision the planner made (so checker can scrutinize it)
- **Primary decision:** use Turbo 2.x's `tasks` key (with `dev` as `persistent`) rather than the legacy `pipeline` key. Rationale: the installed/latest Turbo is 2.x, where `pipeline` is removed; `pipeline` would break `turbo run`. Checker: verify the installed turbo major version matches the schema used, and that `pnpm turbo run build --dry` succeeds.
- **Version pins:** `.nvmrc` = `22.17.0` and `turbo@^2.5.0` are point-in-time picks; if either is not a real published version the executor should substitute the nearest current Node 22 LTS patch / latest stable Turbo 2.x and note it — the constraint (Node 22 LTS, Turbo 2.x) is what matters, not the exact patch.
- **Empty-workspace success semantics:** the plan treats a successful-but-empty `turbo run` (zero matched workspaces) as passing AC2 and the no-op scripts. This matches the card's scoping note. If a future reviewer expects non-empty output they would be wrong for this task stage.
