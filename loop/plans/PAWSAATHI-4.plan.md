## Design decisions the planner locked (scrutinize under "Risks")

1. **Services hub entry point = ONE Settings `ListRow` (`settings-services`) → `/services`; NOT a home quick-action tile.** The home quick-actions grid (`home/quick-actions-grid.tsx`) is a fixed 4-tile contract and pets/[id]'s `QuickActions` feeds the frozen `pet-home` snapshot — inserting a 5th tile is a NEW node (structural), not the additive-only churn the card permits. A Settings `ListRow` is the lowest-blast-radius, honest surface (canon `ListRow`, already dual-themed; Settings has no snapshot; `getByTestId`-based settings tests are unaffected). `app/services.tsx` auto-registers at `/services` via expo-router file discovery (same as `coming-soon`/`weight/[petId]` which are NOT in `app/_layout.tsx`'s explicit `Stack.Screen` list) → **`app/_layout.tsx` is NOT touched.**
2. **No "Notify me at launch", no waitlist, no capture anywhere (HONESTY RULE).** There is no backend to honor a signup, so the mockup's INSURANCE "Notify me at launch" button + "You're on the list!" state is DROPPED entirely. Each service renders a non-interactive informational card carrying a visible **"Coming soon"** badge, plus one honest footer line ("These features aren't available yet."). Cards are `accessible` Views with no `onPress` (a disabled affordance would still imply future action tied to a tap; a plain informational card is the strictly-honest reading).
3. **Services hub introduces ZERO new AA pairs → `dual-theme-contrast.test.ts` gets no new rows.** The hub reuses only §1.1a-verified tokens: text `text-brand-900 dark:text-ink-dark` / `text-brand-700 dark:text-ink-muted-dark`; card `bg-white dark:bg-surface-card-dark`; badge `bg-brand-100 dark:bg-surface-card-dark` + label `text-brand-700 dark:text-accent-bright` (accent-bright-on-card-dark = 5.20 AA, verified). Static local data ⇒ the §6 four-data-states rule (skeleton/error/empty/offline) is **not applicable** (no query); documented, not skipped.
4. **Semantic status fills stay THEME-INVARIANT (frozen, no `dark:`), mirroring PAWSAATHI-3's opaque-fill principle.** The `bg-{amber,green,red}-100` + dark-ink notice blocks (paywall pending/error/restoreNone/success; settings restore success/none/error; `billing-issue-banner` amber) are self-contained opaque LIGHT fills with dark text — legible on ANY OS scheme (a bright box on a dark page, exactly the accepted VetDisclaimer behavior in PAWSAATHI-3). Keeping them byte-identical avoids inventing amber-900/green-900 dark pairs (no new AA math) AND trivially preserves their billing-checklist `testID`s. `billing-issue-banner.tsx` is therefore **ZERO-DIFF**; the paywall/settings notice blocks keep their fill+text classes byte-identical (only surrounding neutral surfaces theme).
5. **Snapshot policy: `pet-home` + `paywall` re-record ADDITIVE-ONLY; `check-result` + `weight-chart` stay BYTE-IDENTICAL.** Theming `pets/[id]` + `PetHeaderCard` + `QuickActions` changes the `pet-home` snapshot's captured `className` literals; theming `paywall.tsx` changes the `paywall` snapshot — both re-recorded and manually verified additive-only. `check-result` and `weight-chart` render no file touched this batch (`weight-chart.tsx` is FROZEN/out-of-scope, `vet-disclaimer.tsx` untouched) → they MUST pass unmodified; any diff ⇒ STOP.
6. **Tab-bar dark treatment = scheme-aware native props (not `dark:` classes), reusing verified tokens.** React-Navigation `tabBar*TintColor`/`tabBarStyle` are runtime props, not className — computed from `useColorScheme()` (§1.6 native-color rule): active `#1f6350`→`#2EA57C` (accent-bright), inactive `#9ca3af`→`#9AA8A1` (ink-muted-dark, 6.50 AA on card-dark), `tabBarStyle` gains `backgroundColor:#16241F` (surface-card-dark) + `borderTopColor:#22392F` (hairline-dark) ONLY in dark; light props stay byte-identical. No new pair.
7. **Fonts (§1.4a) applied matched to each Text's EXISTING weight (never swap visual weight), same as PAWSAATHI-2/3.** `font-display` = screen titles / hero names (`text-2xl/3xl font-bold`); `font-display-semibold` = section/sheet headers (`text-lg font-semibold`); `font-body-semibold` = card titles / list-primary / labels / buttons; `font-body` = body/secondary. Decorative large brand-500 Ionicons (`color="#2f8f74"`, ≥24px) that already-shipped canon components (`empty-state.tsx`, `save-confirmation.tsx`) leave hardcoded STAY hardcoded (they clear the 3:1 UI floor on dark surfaces) — this batch does not churn them.

- **R1 (entry point = Settings only — decision 1, checker WILL probe "why not a home tile"):** a home quick-action tile is a NEW node in the fixed home grid / frozen `pet-home` snapshot (structural, not additive), and adds a 5th tile to a contract the grid tests pin. Cards are non-interactive informational + a visible "Coming soon" badge + "not available yet" note. - **R4 (snapshot re-record — decision 5):** `pet-home`+`paywall` `.snap` change because NativeWind keeps `className` as a literal that now carries `dark:`/`font-*`.
