# Plan — T079: Remote config + A/B scaffold (S)

## Objective (from card)
Extend `GET /v1/config` and its mobile consumer with four things: ① paywall variant assignment
by **stable hash of userId** (env var becomes an override), ② a **min-version gate**
(`minSupportedVersion` in config → mobile launch-time blocking "please update" screen),
③ a **hotline pack version** (`hotlinePackVersion` in config → mobile version-comparison seam for
a future OTA hotline refresh; no download pipeline), and ④ a generalized **`useAppConfig`** hook
with an MMKV-cached last-known-good config (stale-while-revalidate; offline → cache or defaults).
**Accept:** variant assignment is deterministic (unit test); stale-cache offline behavior tested.

## Safety statement (PRODUCT_SPEC §5 / CLAUDE.md §7) — NOT a SAFETY-ESCALATION
- The **only** new blocking surface is the update-gate screen. It **fails OPEN on every
  uncertainty**: `MIN_SUPPORTED_VERSION` defaults to a permissive `"0.0.0"`; a malformed/unparseable
  version on either side ⇒ `isUpdateRequired` returns `false`; a fetch failure/offline ⇒ cached
  config or the permissive default ⇒ not required. So the DEFAULT posture never blocks anyone.
- **Update-gate vs emergency stance (documented per card):** the gate renders only at **cold app
  launch, before any check/intake/red-flag/emergency flow begins** — never mid-check. It is an ops
  lever raised only for clients too broken/unsafe to trust, and even then only when the server's
  `minSupportedVersion` is explicitly **above** the installed version. Because it fails open on all
  uncertainty and is off by default, it cannot silently delay emergency care. The §5 emergency
  escalation code path (`check/emergency/[checkId].tsx`, red-flag rules, hotline rendering) is
  **not touched**.
- **Hotline pack version signals staleness only.** The rendered hotline numbers stay bundled in
  `packages/data` and are **unchanged**; `hotlinePackVersion` never alters displayed emergency/poison
  numbers. The mobile seam is a pure comparison + hook that a future OTA task consumes; no download,
  no mutation of hotline data.
- No `<VetDisclaimer/>`, emergency interstitial, hotline copy, or dosing/`diagnos*` text is
  added or modified. Implementable without weakening any safety surface → proceed (no blocked file).

## Key decisions (each a design choice — scrutinize)
1. **Variant = env override OR stable hash.** `PAYWALL_VARIANT` enum grows to `["A","B","AUTO"]`,
   **default `"AUTO"`**. Semantics (pure fn `assignPaywallVariant(userId, setting)`): if setting is
   `"A"`/`"B"` → force it (override wins); if `"AUTO"` and no userId → `"A"` (stable anonymous
   default); if `"AUTO"` with userId → `fnv1a32(userId) % 2 === 0 ? "A" : "B"`. Hash is a tiny pure
   fnv1a-1a 32-bit function (no dependency). Stateless & deterministic — no per-user persistence.
2. **Auth-optional endpoint via a dedicated `OptionalJwtAuthGuard`, keeping `@Public()`.** The global
   `JwtAuthGuard` (APP_GUARD) short-circuits `true` on `@Public()` without setting `req.user`; a
   controller-scoped `@UseGuards(OptionalJwtAuthGuard)` then runs and does **best-effort** token
   verification — sets `req.user` when a valid Bearer token is present, otherwise leaves it undefined
   and **returns `true` (never 401 / never throws)**. Mobile already calls `/v1/config` with its
   bearer token (via `apiClient`), so the server hashes the real userId; anonymous/onboarding callers
   get the `"AUTO"→A` default. This is the cleanest §6-compliant pattern (a guard, thin controller)
   and reuses the SAME secret-configured `JwtService` (guard added to `AuthModule` providers/exports;
   `RemoteConfigModule` imports `AuthModule`).
3. **Min-version compare is a pure, fail-OPEN function.** `compareVersions(a,b)` parses `x.y.z`
   numeric triples; `isUpdateRequired(current, min)` returns `true` only when both parse AND
   `current < min`; ANY parse failure ⇒ `false`. Mobile reads its own version from
   `Constants.expoConfig?.version` (via `getAppVersion()` in `config.ts`, default `"0.0.0"`).
4. **Hotline pack version = plumbing + seam only.** `hotlinePackVersion` (env int, default `1`) ships
   in config. `packages/data` exports `BUNDLED_HOTLINE_PACK_VERSION` (the version tag of the bundled
   region data — its honest home). Mobile pure fn `isHotlinePackStale(serverVersion, bundledVersion)`
   (`server > bundled`, NaN-safe) + `useHotlinePackStale()` hook = the documented seam a future OTA
   task consumes. No download pipeline, no logging (no `console.log`), no UI now.
5. **`useAppConfig` generalizes `usePaywallConfig` with MMKV cache (SWR).** New
   `app-config-queries.ts` exposes `AppConfig` (`{ variant, minSupportedVersion, hotlinePackVersion }`),
   `DEFAULT_APP_CONFIG` (permissive: `variant "A"`, `minSupportedVersion "0.0.0"`,
   `hotlinePackVersion 1`), `fetchAppConfig()` (GET `/v1/config` → parse with shared schema → on
   success **write MMKV cache** & return; on ANY failure → return **cached** config, else
   `DEFAULT_APP_CONFIG`), and `useAppConfig()` (TanStack Query, `queryKey ["app-config"]`,
   `initialData` = cached-or-default so there is never a loading/undefined config; background refetch
   = stale-while-revalidate). `usePaywallConfig`/`fetchPaywallConfig` are re-implemented as thin
   selectors over the SAME query/fetch (one network call, shared cache) — preserving the
   `PaywallConfig` public shape so `paywall.tsx` is unchanged.
6. **Response schema grows but stays `.strict()` and required.** `appConfigResponseSchema` becomes
   `{ paywall: { variant }, minSupportedVersion: string, hotlinePackVersion: number }`. Server always
   sends all three; the mobile cache re-validates on read (poison-safe, decision in Risks).

## Risks & the decisions the planner made (so checker can scrutinize)
- **R1 — hash distribution.** fnv1a32 `% 2` is roughly, not exactly, 50/50. The distribution test uses
  a loose tolerance band, not an exact split; a real 50/50 sticky bucketing is out of scope (v1 = 2-way).
- **R2 — userId absent skews A.** Anonymous/pre-login callers deterministically get `"A"` (the AUTO
  default). Acceptable: onboarding paywall is one-time and pre-auth; authenticated users (the vast
  majority of paywall views) are hashed. Documented.
- **R3 (SAFETY) — version parse failure fails OPEN, never closed.** `isUpdateRequired` returns `false`
  on any malformed version so a corrupt/odd version string can NEVER lock a user (incl. an emergency
  user) out of the app. This is deliberate.
- **R4 — cache poisoning.** `readCachedConfig` re-validates the MMKV JSON with zod; any invalid/corrupt
  stored value is treated as no cache (→ defaults), so a bad cache can neither force an update gate nor
  inject a bogus variant/version.
- **R5 — optional guard fail-open.** An invalid/expired token on `/v1/config` is treated as anonymous
  (no 401, variant falls back to default) rather than erroring — correct for a `@Public` config
  endpoint; verified by spec + e2e.
- **R6 — `PAYWALL_VARIANT` default changed `A → AUTO`.** Dev/default behavior for authenticated users
  becomes hash-based (the whole point of the A/B scaffold). Unauthed default output is still `"A"`
  (AUTO + no user), so the existing e2e's variant expectation is preserved modulo the two added fields.
- **R7 — iOS store URL provisional.** No App Store numeric app id exists pre-submission (confirmed at
  C3 after T102). `IOS_UPDATE_URL` is a documented provisional (marketing/site link) with a
  `// NOTE(T102):` reference; Android uses the real package-scoped Play URL. Bundle id
  `com.pawcareright.app` is safe to hardcode (§1a — identifier, not display name).
- **R8 — `_layout` refactor.** `useAppConfig` must run inside `PersistedApiQueryProvider`; the
  `<UpdateGate>` wrapper is placed inside the provider around the Stack. The `restoring` splash branch
  is left untouched so startup ordering is unchanged.
- **R9 — hotline seam is intentionally inert.** `useHotlinePackStale` is exported + tested but not yet
  rendered/consumed (the OTA refresh is a later task). This is the documented seam, not dead code — no
  `console.log`, no fake UI.
- **R10 — data-package barrel.** If `packages/data`'s public barrel enumerates region exports,
  `BUNDLED_HOTLINE_PACK_VERSION` must be added there too; otherwise the regions `index.ts` export
  suffices. Executor confirms before adding.
