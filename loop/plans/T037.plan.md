## Safety statement
This task authors the PRODUCT_SPEC §5 **measurement corpus** — it changes NO §5 surface. It ships no disclaimer, emergency-escalation, or dosing code; it only adds fixture YAML + one jest spec. Tier assignments in the golden set are themselves safety content and are the C1 vet-review artifact. ## Authoring mechanics (READ FIRST — the whole task is arithmetic)

### Severity ladder (from `@pawcareright/types`)
`EMERGENCY_NOW(0) · VET_24H(1) · VET_SOON(2) · MONITOR(3) · REASSURE(4)` — lower = more urgent.

### Pipeline that scores each case (fixed by T036, DO NOT change)
`evaluateRedFlags(input)` → `rulesFloor = highest?.tierFloor ?? null` (every rule floors to `EMERGENCY_NOW`) · `runTriage(fakeResponse)` → `aiResult` · `applyPostRules(aiResult,{species,rulesFloor})` → `finalTier`,`source`. Steps only ever RAISE urgency, in order:
1. **confidence floor**: `low` + tier less urgent than `VET_SOON` → `VET_SOON` (already guaranteed by `parseTriage`).
2. **cat bias**: `species==="CAT"` → `raiseOne(tier)` (one step MORE urgent; no-op at `EMERGENCY_NOW`).
3. **rules floor**: `moreUrgent(tier, rulesFloor)`; `source="rules"` **iff this step changed the tier**, else `"ai"`.
4. **homeCare strip**: `homeCare=[]` when final tier ∉ {VET_SOON,MONITOR,REASSURE}.

### `fakeResponse` → FINAL tier table (THE core of authoring)
| Target FINAL tier | DOG `fakeResponse.urgency` | CAT `fakeResponse.urgency` |
|---|---|---|
| EMERGENCY_NOW (AI-driven) | `EMERGENCY_NOW` | `VET_24H` (bias→EMERGENCY) |
| VET_24H | `VET_24H` | `VET_SOON` (bias→VET_24H) |
| VET_SOON | `VET_SOON` | `MONITOR` (bias→VET_SOON) |
| MONITOR | `MONITOR` | `REASSURE` (bias→MONITOR) |
| REASSURE | `REASSURE` | **IMPOSSIBLE** |

**The trap, resolved (Risk R1):** cat bias is `raiseOne`, which only *increases* urgency; there is no AI tier that maps DOWN to `REASSURE` for a cat (`raiseOne(REASSURE)=MONITOR`). Therefore **a cat can never land REASSURE via the AI path.** Decision: the **REASSURE tier is DOG-ONLY** (`expectedTier: REASSURE` cases are all dogs). Cat coverage in the least-urgent band lives in the MONITOR file (cats reach MONITOR from AI `REASSURE`). The meta-test's REASSURE-species minimum is `dog≥30, cat exempt (0 allowed)`.

### `parseTriage`-validity rules for every `fakeResponse` (else the case silently becomes SAFE_FALLBACK/VET_SOON and fails)
- `confidence:"low"` is **only** legal with urgency ∈ {EMERGENCY_NOW, VET_24H, VET_SOON}. Use `medium`/`high` for MONITOR/REASSURE fake responses.
- `homeCare` MUST be `[]` when `fakeResponse.urgency` ∈ {EMERGENCY_NOW, VET_24H}; it may be non-empty only for VET_SOON/MONITOR/REASSURE.
- No `/diagnos/i` substring anywhere; `possibleCauses` ≤ 4; each string ≤ 400 chars, ≤ 10 per array; `summary` 1–600 chars; `followUpHours` = positive int ≤ 2160 **or** `null`.
- **Detector-clean (T4):** no `\d+\s*(mg|ml|mcg|g|kg|iu|tsp|tbsp)`, no `mg/kg`, no `per kg|per pound|per lb` in any user-facing string. Keep numbers/units out of `summary`/`doNot`/etc. (numeric `followUpHours` is a number field, not scanned — fine).

### Rules-floor pattern (emergency-via-rules cases)
- Set `expectedTier: EMERGENCY_NOW`, `expectSource: rules`, `expectRedFlagRule: <ruleId>`, and **OMIT `fakeResponse`** (default = valid `SAFE_FALLBACK` = VET_SOON/low). Then: dog → rules floor raises VET_SOON→EMERGENCY (`source=rules`); cat → bias VET_SOON→VET_24H → rules floor →EMERGENCY (`source=rules`). Uniform and provider-independent — this is exactly "provider not consulted for the tier floor."
- **Fire the rule via `signs` (structured booleans), not free text** — deterministic and avoids keyword false-negatives. `freeText` is optional owner-voice colour and must NOT introduce dosing/diagnosis language.
- **Source-must-stay-`rules`**: never let the fake (after bias) already reach EMERGENCY (that flips `source` to `ai`). OMITting `fakeResponse` guarantees this for both species. Do NOT give rules cases an explicit EMERGENCY/VET_24H fake.
- Isolation gotchas baked into the blueprint below: rule 19 (`urinary-obstruction-signs-cat`) uses `sex: FEMALE` so rule 3 (`urinary-blockage-cat`, sex gate MALE/UNKNOWN) does NOT also fire; rule 18 (`distended-abdomen`) uses `distended_abdomen` alone (no retching) so rule 2 (`gdv-suspected`, DOG+both) is not the target. `expectRedFlagRule` only needs to be *among* `matched` (it is), so incidental co-matches are harmless.

## Meta-test spec — packages/ai/src/evals/golden-set.spec.ts (create)
Imports: `loadCases` (`./load`), `evalsDir` (`./paths`), `RED_FLAG_RULES`,`evaluateRedFlags` (`../rules`), `applyPostRules` (`../post-rules`), `parseTriage`,`SAFE_FALLBACK`,`URGENCY_SEVERITY` (`@pawcareright/types`). Load once: `const golden = loadCases(evalsDir()).filter(c => c.set === "golden")`. Build `RedFlagIntake` from `c.input` (species,sex,ageMonths,weightKg,sizeClass,signs,freeText) inline. Primary tier helper: `c.expectedTier ?? c.acceptableTiers![0]`. Assertions:
1. `golden.length >= 150`.
2. Unique ids (`loadCases` already throws on dupes; also assert `new Set(ids).size === ids.length`).
3. Per primary tier count ≥ 30 for each of the 5 tiers.
4. Per-tier species minimums: EMERGENCY_NOW/VET_24H/VET_SOON/MONITOR → dog ≥ 12 AND cat ≥ 12; REASSURE → dog ≥ 30 (cat exempt; assert cat REASSURE count === 0 to lock the R1 decision).
5. Overall species: `dog/total ≥ 0.40` AND `cat/total ≥ 0.40` (over all golden incl. samples).
6. Rule coverage: the set of all `expectRedFlagRule` values ⊇ every `RED_FLAG_RULES[*].id` (all 22).
7. For every case with `expectRedFlagRule`: `expectSource === "rules"`, `expectedTier === "EMERGENCY_NOW"`, and `evaluateRedFlags(intake)` has `highest?.tierFloor === "EMERGENCY_NOW"` and `matched.some(m => m.ruleId === expectRedFlagRule)`.
8. For every case with `expectSource === "ai"`: `evaluateRedFlags(intake).matched.length === 0` (no rule fires → `rulesFloor` null → source stays `ai`). Guards accidental keyword hits in `freeText`.
9. Age extremes: `count(ageMonths <= 6) >= 15` AND `count(ageMonths >= 120) >= 15`.
10. Ambiguity: `count(c.acceptableTiers !== undefined) >= 15`.
11. Every `fakeResponse` (when present) → `parseTriage(fakeResponse).ok === true`. Catches low+low-tier, homeCare-on-emergency, diagnos word, >4 causes, bad followUpHours.
12. **Deterministic finalTier pre-check** (mirrors the pipeline sans provider): for each case, `parsed = c.fakeResponse ? parseTriage(c.fakeResponse).result : SAFE_FALLBACK`; `outcome = applyPostRules(parsed, {species: c.input.species, rulesFloor: evaluateRedFlags(intake).highest?.tierFloor ?? null})`; assert `outcome.finalTier ∈ (c.acceptableTiers ?? [c.expectedTier])`, and when `expectSource` set assert `outcome.source === c.expectSource`. (Fast local proof before the harness run; the harness run is authoritative — if they ever diverge, fix the case.)

## Risks & design decisions (numbered — for checker scrutiny)
1. **R1 — REASSURE is DOG-ONLY (the task's trap).** Cat bias `raiseOne` only increases urgency; no AI tier maps to REASSURE for a cat (`raiseOne(REASSURE)=MONITOR`). So `expectedTier:REASSURE` cats are impossible via the AI path. Decision: REASSURE tier = 30 dogs; cat least-urgent coverage lives in MONITOR (cats reach MONITOR from AI `REASSURE`). Meta-test locks this (`cat REASSURE count === 0`). This satisfies "≥30 per tier across dogs+cats" as ≥30-per-tier with the documented species exception the card's own "acceptable-tier ranges" language permits.
2. **R2 — rules cases OMIT `fakeResponse` (rely on SAFE_FALLBACK).** Uniform, provider-independent, and guarantees `source==="rules"` for BOTH species (dog VET_SOON→EMERGENCY; cat VET_SOON→bias VET_24H→EMERGENCY) without risking a fake that accidentally reaches EMERGENCY and flips `source` to `ai`. Trade-off: these cases don't demonstrate the floor beating a *concrete non-fallback* under-call — but samples 1–2 already do, and assertion 12 proves the floor holds from SAFE_FALLBACK. Executor may optionally add an explicit under-calling fake (MONITOR dog / VET_SOON cat) to a few rules cases for the C1 story, but MUST keep it strictly less urgent than EMERGENCY after bias.
3. **R3 — counting convention = `expectedTier ?? acceptableTiers[0]` ("primary tier").** Deterministic and file-aligned (every ambiguity case's `acceptableTiers[0]` equals its file's tier). Alternative (most-urgent acceptable tier) would misfile ambiguity cases; rejected.
4. **R4 — reusable per-tier `fakeResponse` templates.** Reusing 9 canonical detector-clean, `parseTriage`-valid JSON skeletons across ~120 AI cases eliminates the dominant authoring risk (150 hand-written JSONs) while keeping each case's scoring independent. Only `id`+`description` must differ; summary wording may vary for realism. Assertion 11 + the harness detector still validate every one.
5. **R5 — fire rules via `signs`, not free text.** Structured booleans fire deterministically regardless of the keyword normalizer; `freeText` is optional colour. Removes the biggest false-negative risk in emergency coverage. Isolation choices (rule 19 `sex:FEMALE`; rule 18 `distended_abdomen` alone) are baked into the blueprint and justified inline.
6. **R6 — species split lands cat at ~40.9% overall.** Tight but ≥40%; `vet-24h.yaml` is 14 dog/16 cat to buy margin. If the checker prefers more headroom, shifting 1–2 MONITOR/VET_SOON dogs to cats is a safe in-plan tweak (still ≥12 dog/tier). Not doing so preserves clean 15/15 files.
7. **R7 — meta-test assertion 12 re-derives finalTier locally.** It duplicates the pipeline's post-rules math (minus `runTriage`, which returns a valid fake unchanged). Justified: it catches cat-bias arithmetic errors in-unit (fast, per-case) before the slower end-to-end `pnpm test:ai-evals`. The harness run remains authoritative; on any divergence the case is fixed, not the assertion.
