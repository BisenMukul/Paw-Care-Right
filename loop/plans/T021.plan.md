# Plan — T021: Pet model + CRUD API

## Objective (from card)
Add the `Pet` domain (ARCHITECTURE §3) with household-scoped CRUD under flat `/v1/pets` routes, soft-delete (`deletedAt`) with default-filtered queries, and integration tests proving cross-household 404, soft-delete hiding, and the age-XOR-birthDate validation matrix.

## Key decisions (all flagged for the checker under "Risks & Decisions")
- **Flat routes + guard "resolve-from-membership" mode (Option B).** Keep ARCHITECTURE §4's flat `/pets`, `/pets/:id`; extend the household-scope machinery with a second, additive metadata mode that resolves the caller's SOLE membership (v1 = one household/user). Param mode is untouched → all existing guard assertions stay green.
- **Pet schemas live in `packages/types` (Zod)** + api class-validator DTOs; enums tied three-way (see Risk R1).
- **XOR enforced in the service** against the effective post-merge state (not the DTO), so PATCH is handled deterministically.
- **DELETE requires OWNER**; create/update/get/list allow any member.
- **Single composite index `@@index([householdId, deletedAt])`** (leading `householdId` also serves the FK lookup).
- **Pagination deferred** — `GET /pets` returns a plain array (card does not ask; §2 rule 2).

## Interfaces/contracts (executor must match)

### Prisma `Pet` model
```prisma
model Pet {
  id                String    @id @default(uuid())
  householdId       String
  species           Species
  breedSlug         String?
  name              String
  sex               Sex       @default(UNKNOWN)
  neutered          Boolean   @default(false)
  birthDate         DateTime?
  ageEstimateMonths Int?
  weightGrams       Int?
  photoKey          String?
  deletedAt         DateTime?
  createdAt         DateTime  @default(now())
  updatedAt         DateTime  @updatedAt
  household         Household @relation(fields: [householdId], references: [id], onDelete: Cascade)

  @@index([householdId, deletedAt])
}
```
Add to `Household`: `pets Pet[]`.

### packages/types (Zod)
- `SPECIES = ["DOG","CAT"] as const`, `SEX = ["MALE","FEMALE","UNKNOWN"] as const`.
- `createPetSchema`: `species` (required), `name` (non-empty string), `sex` (optional, default `UNKNOWN`), `neutered` (optional bool), `breedSlug`/`photoKey` (optional string), `birthDate` (optional ISO date string), `ageEstimateMonths`/`weightGrams` (optional positive int). `.superRefine` → error if both `birthDate` and `ageEstimateMonths` are present & non-null.
- `updatePetSchema = createPetSchema.partial()` with fields allowing `null`.
- `petSchema`: id (`petIdSchema`), householdId, species, sex, name, neutered, nullable breedSlug/birthDate/ageEstimateMonths/weightGrams/photoKey, createdAt, updatedAt. (No `deletedAt` — internal only.)

### DTOs (class-validator, tied to Prisma enums)
- `CreatePetDto`: `@IsEnum(Species) species`; `@IsString()@IsNotEmpty() name`; `@IsOptional()@IsEnum(Sex) sex`; `@IsOptional()@IsBoolean() neutered`; `@IsOptional()@IsString() breedSlug`/`photoKey`; `@IsOptional()@IsDateString() birthDate`; `@IsOptional()@IsInt()@Min(0) ageEstimateMonths`/`weightGrams`. `@ApiProperty` on each. (Enum-arrays for Swagger from `Object.values(Species)`/`Object.values(Sex)`.) **No XOR validator here** — XOR is service-level.
- `UpdatePetDto`: same fields, all `@IsOptional()`, nullable where clearing is allowed (use `@ValidateIf((_,v)=>v!==null)` or `@IsOptional()` semantics so explicit `null` is accepted to clear `birthDate`/`ageEstimateMonths`/etc.).

### Service (`PetsService`)
- `PetResponse` interface (public shape mirroring `petSchema`; maps Prisma row → response, drops `deletedAt`).
- `create(householdId, dto): Promise<PetResponse>` — assert XOR on `{birthDate, ageEstimateMonths}`; create with `householdId`.
- `findAll(householdId): Promise<PetResponse[]>` — `where { householdId, deletedAt: null }`, ordered `createdAt desc`.
- `findOne(householdId, id): Promise<PetResponse>` — `findFirst where { id, householdId, deletedAt: null }`; null → `NotFoundException`.
- `update(householdId, id, dto): Promise<PetResponse>` — load scoped row (404 if absent); compute effective `birthDate`/`ageEstimateMonths` = `key in dto ? dto[key] : existing[key]` (explicit `null` clears); assert XOR on the effective pair; persist only provided keys.
- `remove(householdId, id): Promise<PetResponse>` — scoped `findFirst` (404 if absent/already-deleted); set `deletedAt = new Date()`; return the soft-deleted resource.
- Private `assertAgeXor(birthDate, ageEstimateMonths)` → throws `BadRequestException` (→ 400 `VALIDATION_FAILED`) when both are non-null.

### Controller (`PetsController`, all under global `/v1` prefix)
- `@HouseholdFromMembership()` on the controller class (or each handler) + `@CurrentHousehold()` to read `scope.householdId`.
- `GET /pets` → `findAll` → 200 array.
- `POST /pets` → `create` → 201.
- `GET /pets/:id` → `findOne` → 200.
- `PATCH /pets/:id` → `update` → 200.
- `DELETE /pets/:id` → `@RequireRole("OWNER")` → `remove` → 200 (returns the soft-deleted pet).
- Swagger response decorators on every handler.

### Guard extension (`household-scope.guard.ts`)
Additive, order-preserving:
```
paramName = reflector.getAllAndOverride(HOUSEHOLD_SCOPE_PARAM_KEY, ...)
if (paramName !== undefined) { /* EXISTING path — unchanged */ }
else {
  fromMembership = reflector.getAllAndOverride(HOUSEHOLD_SCOPE_FROM_MEMBERSHIP_KEY, ...)
  if (!fromMembership) return true          // no-op (unchanged behavior)
  if (!userId) throw UnauthorizedException
  memberships = prisma.membership.findMany({ where: { userId } })
  if (memberships.length === 1) { inject scope; return true }
  throw NotFoundException                    // 0 memberships → 404; >1 → 404 (v1 defer, see R2)
}
```

## Tests to write (map to acceptance criteria)

### AC: cross-household 404 (flat routes)
- `pets.e2e-spec` → "user B GET /pets/:id of user A's pet → 404 NOT_FOUND" (also PATCH → 404, DELETE → 404). Asserts the pet is only ever fetched scoped to the resolved household.
- `pets.service.spec` → `findOne`/`update`/`remove` always call Prisma with `where` including `householdId` + `deletedAt: null`; wrong household → `NotFoundException`.

### AC: soft-delete hides from list (and direct get)
- `pets.e2e-spec` → "create → DELETE (owner) → GET /pets omits it AND GET /pets/:id → 404".
- `pets.service.spec` → `remove` writes `deletedAt`; `findAll`/`findOne` include `deletedAt: null` in `where`; a row with `deletedAt` set is absent.

### AC: validation — age XOR birthDate allowed states
- `pets.e2e-spec` matrix on POST: both set → 400 `VALIDATION_FAILED`; birthDate only → 201; ageEstimateMonths only → 201; neither → 201.
- `pets.e2e-spec` PATCH: pet with birthDate, PATCH adds `ageEstimateMonths` without nulling → 400; PATCH sets `ageEstimateMonths` AND `birthDate: null` in one body → 200.
- `pets.service.spec` → `assertAgeXor` throws on both-set; `update` merges effective state before asserting.

### Supporting (module correctness / regressions)
- `pets.e2e-spec` → happy path create→get→list→patch; DELETE role (member → 403 FORBIDDEN, owner → 200); unauthenticated → 401; a bare user with **no household** GET /pets → 404 (resolve-from-membership no-membership path).
- `pet.spec` (types) → species/sex enums accept valid & reject invalid; `createPetSchema` XOR superRefine rejects both-set; `updatePetSchema` partial accepts empty object.
- `household-scope.guard.spec` → from-membership mode: exactly-one membership injects scope & returns true; zero → `NotFoundException`; >1 → `NotFoundException` (documented v1 defer); missing user → `UnauthorizedException`; both metadata absent → returns true (no-op non-regression).
- `pets.service.spec` enum-sync assertion → `speciesSchema.options` deep-equals `Object.values(Species)` and `sexSchema.options` deep-equals `Object.values(Sex)` (mechanically fails CI on Prisma↔Zod drift — this is how the checker verifies R1 sync).

## Risks & Decisions (scrutinize)
- **R1 — Three-way enum representation (Prisma / Zod / DTO).** Prisma `Species`/`Sex` are the DB truth; Zod mirrors the literals for shared clients; DTOs validate with `@IsEnum(Species|Sex)` (DTO↔Prisma tie). Prisma↔Zod drift is caught by the enum-sync assertion test (`pets.service.spec`). Dual-maintenance risk accepted; the failing-on-drift test is the guardrail. (packages/types must NOT import `@prisma/client`, so the sync test lives in the api.)
- **R2 — Guard resolve-from-membership treats >1 membership as 404.** Correct for v1 (exactly one household/user, auto-provisioned). When multi-household lands (T027/28) this mode needs a selector; a code comment + this risk flag mark it. It fails safe (denies), never leaks/over-grants.
- **R3 — XOR lives only in the service (not the DTO).** A DTO-level validator can't see existing state for PATCH, so a single service-level `assertAgeXor` against the effective merged state is the deterministic rule: explicit `null` in a PATCH clears a field; both effectively non-null → 400. Zod `createPetSchema` also refines both-set for client UX, but the server service is authoritative.
- **R4 — DELETE = OWNER, others = any member.** Matches ARCHITECTURE's "role guard OWNER for destructive ops"; reuses `@RequireRole("OWNER")`. If product later wants members to delete, revisit.
- **R5 — Single composite `@@index([householdId, deletedAt])`.** Serves both the FK lookup (leading column) and the default-filtered list; a separate `@@index([householdId])` would be redundant (over-indexing). `@Index` choice flagged for review against §6's "every FK / new query pattern indexed".
- **R6 — Household→Pet `onDelete: Cascade`.** Deleting a household hard-deletes its pets (pets die with the household); this also keeps `cleanupUsers` working without a factory change. Alternative (Restrict) rejected — a household has no meaning without its owner/pets.
- **R7 — DELETE returns 200 + the soft-deleted resource** (not 204) so the e2e can assert `deletedAt` is set; minor REST-style choice.
- **Safety:** none — pure domain CRUD, no AI output, disclaimers, emergency, or dosing surface touched. Not a SAFETY-ESCALATION.
