## Interfaces/contracts (executor must match these shapes)
`src/error-codes.ts`:
```ts
import { z } from "zod";
export const errorCodeSchema = z.enum([
  "VALIDATION_FAILED", "UNAUTHORIZED", "FORBIDDEN", "NOT_FOUND",
  "CONFLICT", "RATE_LIMITED", "PAYMENT_REQUIRED", "INTERNAL",
]);
export type ErrorCode = z.infer<typeof errorCodeSchema>;
export const errorResponseSchema = z.object({
  error: z.object({
    code: errorCodeSchema,
    message: z.string(),
    requestId: z.string(),
  }),
});
export type ErrorResponse = z.infer<typeof errorResponseSchema>;
```
`src/pagination.ts` (cursor convention `?cursor=&limit=`, ARCHITECTURE.md L82):
```ts
export const paginationQuerySchema = z.object({
  cursor: z.string().optional(),
  limit: z.coerce.number().int().min(1).max(100).default(20),
});
export type PaginationQuery = z.infer<typeof paginationQuerySchema>;
export function paginated<T extends z.ZodTypeAny>(item: T) {
  return z.object({ items: z.array(item), nextCursor: z.string().nullable() });
}
```
`src/branded-ids.ts`:
```ts
export function brandedId<B extends string>(brand: B) {
  return z.string().uuid().brand<B>();
}
export type Branded<B extends string> = z.infer<ReturnType<typeof brandedId<B>>>;
export const petIdSchema = brandedId("PetId");
export type PetId = z.infer<typeof petIdSchema>;
```

## Risks & the design decisions the planner made (checker: scrutinize each)
- **R1 — Dual ESM+CJS output.** tsup `format: ["esm","cjs"]` + `dts`. Rationale: cheap, and the T006 NestJS consumer is CJS-leaning; ESM-only would risk a require() interop failure later. Cost: two dist artifacts. Alternative (ESM-only) rejected as future-fragile.
- **R2 — exports/consumption always via dist.** No `development`/source condition; dev-time consumers read `dist` too. Safe because turbo `build.dependsOn: ["^build"]` orders the types build before any dependent's typecheck/test/lint. Consequence: the AC2 scratch typecheck REQUIRES `pnpm build` to have run first (Step 7 before Step 9) — called out explicitly because Sonnet may otherwise typecheck against a missing dist.
- **R3 — Starter error-code set.** Seeded 8 codes (VALIDATION_FAILED, UNAUTHORIZED, FORBIDDEN, NOT_FOUND, CONFLICT, RATE_LIMITED, PAYMENT_REQUIRED, INTERNAL). This is a minimal enum that later tasks extend; not exhaustive by design.
- **R4 — Modifying `tsconfig.base.json` (a T002-owned file).** The card explicitly says "consumed path aliases wired in tsconfig base", which authorizes this single edit. Decision: add `baseUrl: "."` and `paths: { "@pawcareright/types": ["../types/dist/index.d.ts"] }` pointing at the built declaration (consistent with the always-dist decision R2). Interaction risk: the shared base is extended by every workspace (incl. config itself); `paths`/`baseUrl` resolve relative to the base file's directory (`packages/config`), so `../types/dist` is correct from every extender. pnpm workspace resolution already resolves the import via node_modules symlink, so the alias is somewhat redundant but card-mandated; it must not break config's own typecheck (config imports nothing from types). If this addition breaks any existing typecheck, the executor should STOP and flag rather than removing other base options.
- **R5 — New dependency `tsup`.** Card-mandated build tool; actively maintained esbuild-based bundler, de-facto standard for TS libs. Requires a one-line justification in `loop/journal.md` (CLAUDE.md §2.7) — flag for the orchestrator; the executor does not journal.
- **R6 — Scratch cleanup discipline.** The `.scratch-consumers/` tree is transient; if it survives into the diff the scope/quality gate should fail. Deletion (Step 10) is mandatory. Placed inside `packages/types/` so the import resolves via the workspace self-symlink under NodeNext + Bundler.
- **Safety:** T005 touches no Safety Policy surface (no disclaimers, emergency escalation, or dosing copy) — pure type/schema scaffolding. No SAFETY-ESCALATION.
