### AC2 — "CTA visible above the fold on small devices (layout assertion)"

Two complementary headless assertions in `pet-home-screen.test.tsx`, both in the online loaded state (no offline banner):

- **(a) Structural — CTA is pinned outside/above the scroll region.** Render; get `pet-home-scroll`; assert `within(scroll).queryByTestId("pet-home-cta")` is `null` (CTA is NOT inside the scrollable subtree) AND root `getByTestId("pet-home-cta")` resolves; assert `pet-home-header-region` (which contains the CTA) appears BEFORE `pet-home-scroll` in the parent's children order. Proves the CTA can never be scrolled off the top — it is fixed above the scrollable content.
- **(b) Budget — fixed heights fit iPhone SE.** Mock `Dimensions.get("window")` → `{width:320,height:568}`. Import the layout constants and assert `ABOVE_FOLD_BUDGET <= SE_WINDOW.height`. To bind the arithmetic to the ACTUAL UI (non-vacuity), also assert the rendered nodes carry those exact constants as applied styles: `pet-home-header-card` style height === `HEADER_CARD_HEIGHT` and `pet-home-cta` style minHeight === `CTA_HEIGHT` (flatten style). Optionally drive `onLayout` on `pet-home-header-region` and `pet-home-cta` with the same constant heights via `fireEvent(node,"layout",{nativeEvent:{layout:{height:CONST}}})` to mirror the on-device path.

**Non-vacuity argument (state explicitly for the checker):** the budget test cannot pass by asserting an arbitrary literal, because the same constants it sums are the ones asserted to be the header-card's and CTA's *applied* style dimensions (step-(b) style checks) and are what the component actually lays out (structural step-(a) guarantees the CTA sits in the pinned region above the scroll). If the executor inflates a constant to force the fold, the style-equality assertions still hold but `ABOVE_FOLD_BUDGET <= 568` breaks; if it shrinks the applied style away from the constant, the style-equality assertions break. The two assertions can only be simultaneously satisfied by a layout that genuinely fits. **Limitation:** this is a fixed-height simulation, not a real Yoga layout pass (RNTL has no native layout engine); true on-device above-fold is re-confirmed in the M2 manual checklist. Documented as Risk R3.

## Interfaces / contracts

### `src/pets/pet-age.ts`
```ts
import type { Pet } from "@pawcareright/types";
// Uses birthDate (ISO string|null) and ageEstimateMonths (int|null).
export function derivePetAgeLabel(
  pet: Pick<Pet, "birthDate" | "ageEstimateMonths">,
  now?: Date, // defaults to new Date(); passed explicitly by tests/snapshots
): string;
```
Algorithm: if `birthDate` set → totalMonths = whole calendar months between birthDate and now, clamped ≥ 0 (decrement one month when now's day-of-month < birthDate's day-of-month). `< 12` → `` `${m} ${age.mo}` ``; else years=`floor(total/12)`, rem=`total%12`; rem===0 → `` `${years} ${age.yr}` `` else `` `${years} ${age.yr} ${rem} ${age.mo}` ``. Else if `ageEstimateMonths` set → `< 12` → `` `${age.approx}${m} ${age.mo}` `` else `` `${age.approx}${floor(m/12)} ${age.yr}` `` (residual months dropped — approximate). Else → `age.unknown`.

### `src/pets/pet-home-layout.ts`
```ts
export const SE_WINDOW = { width: 320, height: 568 } as const; // iPhone SE
export const SAFE_AREA_TOP_RESERVE = 44;   // status bar / notch reserve, SE-class
export const HEADER_CARD_HEIGHT = 140;     // photo + name + age + breed, fixed
export const SECTION_GAP = 16;
export const CTA_HEIGHT = 56;
// Top-of-screen through bottom of the CTA:
export const ABOVE_FOLD_BUDGET =
  SAFE_AREA_TOP_RESERVE + HEADER_CARD_HEIGHT + SECTION_GAP + CTA_HEIGHT; // = 256 ≤ 568
```
(Executor may tune the four leaf numbers, but `ABOVE_FOLD_BUDGET <= SE_WINDOW.height` must hold and each leaf must be the value actually applied in the corresponding style.)

## Risks & the decisions the planner made (for the checker to scrutinize)
- **R1 (photo source).** No display-URL endpoint exists, so the header photo uses only the `localPhoto` route param (present right after the wizard) and otherwise shows a species placeholder. Decision, not an omission.
- **R2 (offline model).** Chose banner-over-cache when `usePet` has cached `pet` data (non-blocking `pet-home-offline-banner`), and a full-screen `pet-home-offline` state only when there is no data. The above-fold budget test runs in the online loaded state, so the banner is excluded from the fold budget.
- **R3 (above-fold is a simulation).** RNTL has no Yoga layout pass; the assertion combines a structural pin (CTA outside/above the ScrollView) with a fixed-height arithmetic budget bound to the actually-applied style constants. It cannot detect real wrapping/overflow — on-device confirmation folds into the M2 manual check.
- **R4 (layout constant values).** The four leaf heights (44/140/16/56 → 256) are chosen estimates; correctness of the test rests on them matching the applied styles, enforced by the style-equality assertions in AC2(b).
- **R5 (empty vs error).** Distinguished by `usePet` returning `null`/`undefined` (empty) vs `isError` (error). If the API returns 404-as-throw, "not found" surfaces as error, not empty — acceptable for this stub screen; noted for the checker.
- **R6 (age approximation).** For `ageEstimateMonths ≥ 12` the residual months are dropped ("~X yr"), per the card's approximate display intent.
- **R7 (snapshot determinism).** Age depends on `now`; snapshots freeze time via `jest.useFakeTimers({now: FIXED_MS})` so the derived age label is stable.

Safety: this screen adds no AI output, no diagnosis, no dosing, and does not touch any disclaimer/emergency surface — the "Something wrong?" CTA is an inert stub with neutral copy. No Safety Policy surface is weakened (PRODUCT_SPEC §5 / CLAUDE.md §7). No SAFETY-ESCALATION.
