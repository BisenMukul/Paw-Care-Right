# Plan — T029: Provider abstraction (packages/ai seam)

## Objective (from card)
Create the greenfield `@pawcareright/ai` package: the `TextProvider`/`VisionProvider`/`ImageProvider` interfaces, a `registry.ts` that selects impls from env, real Ollama Cloud text+vision impls (OpenAI-compatible `/v1/chat/completions`) and a Gemini image impl (REST `generateContent`), deterministic `fake/` providers for tests/CI, and cost/latency capture — all via plain `fetch`, **no vendor SDK, no new external dep, zero `ANTHROPIC_API_KEY`**. Pure infrastructure seam only: NO prompts, triage schema, parsing, disclaimer, or fallback logic (those are T030–T033).

Safety: T029 touches NO Safety-Policy surface (PRODUCT_SPEC §5 / CLAUDE §7) — no disclaimer, emergency interstitial, triage/food output, dosing, or med copy. The providers deliberately return **raw** text/usage; validation, `parseTriage`, repair-retry, and SAFE_FALLBACK are downstream (AI_PROVIDERS §3, owned by T030/T033). Implementing any of that here is OUT OF SCOPE and would create an unreviewed safety surface. This is NOT a safety escalation.

## Interfaces/contracts (executor must match)
```ts
// providers/types.ts
export const DEFAULT_TIMEOUT_MS = 30_000;

export interface ProviderUsage {
  latencyMs: number;               // ALWAYS measured
  inputTokens?: number;            // from provider usage when present
  outputTokens?: number;
  totalTokens?: number;
  costMicroUsd?: number;           // computed from tokens × CostRates (default 0)
}

export interface TextMessage { role: "system" | "user" | "assistant"; content: string; }
export interface TextGenerateOptions {
  prompt?: string;                 // one of prompt | messages required
  messages?: TextMessage[];
  system?: string;
  temperature?: number;            // default 0 (triage determinism)
  timeoutMs?: number;              // default DEFAULT_TIMEOUT_MS
  maxTokens?: number;
}
export interface TextResult { text: string; model: string; usage: ProviderUsage; raw?: unknown; }
export interface TextProvider { generate(options: TextGenerateOptions): Promise<TextResult>; }

export interface VisionImage { base64?: string; url?: string; mimeType: string; }
export interface VisionGenerateOptions {
  prompt: string; images: VisionImage[]; system?: string;
  temperature?: number; timeoutMs?: number; maxTokens?: number;
}
export interface VisionProvider { generate(options: VisionGenerateOptions): Promise<TextResult>; }

export interface ImageGenerateOptions { prompt: string; size?: string; timeoutMs?: number; }
export interface ImageResult { imageBase64?: string; url?: string; model: string; usage: ProviderUsage; raw?: unknown; }
export interface ImageProvider { generateImage(options: ImageGenerateOptions): Promise<ImageResult>; }

export type ProviderErrorCode = "timeout" | "http_error" | "invalid_response" | "config_error";
export class ProviderError extends Error {
  readonly code: ProviderErrorCode;
  readonly provider: string;
  readonly status?: number;
  constructor(provider: string, code: ProviderErrorCode, message: string, status?: number);
}
```

// usage.ts
export interface CostRates { costPerMInputUsd?: number; costPerMOutputUsd?: number; }

}): ProviderUsage;   // totalTokens = in+out; costMicroUsd from rates (default 0)

Ollama request (text): `POST ${OLLAMA_CLOUD_BASE_URL}/v1/chat/completions`, headers `{ Authorization: "Bearer "+OLLAMA_CLOUD_API_KEY, "Content-Type": "application/json" }`, body `{ model: AI_TEXT_MODEL, messages, temperature: 0, stream: false, max_tokens? }` (system prepended as a `{role:"system"}` message). Response map: `choices[0].message.content`→text; `usage.prompt_tokens`→inputTokens, `usage.completion_tokens`→outputTokens; `model` from response `model` ?? env.
Ollama vision: same, `messages:[{ role:"user", content:[ {type:"text",text:prompt}, {type:"image_url", image_url:{ url }} … ] }]` with `url = base64 ? "data:"+mimeType+";base64,"+base64 : url`.
Gemini: `POST https://generativelanguage.googleapis.com/v1beta/models/${GEMINI_IMAGE_MODEL}:generateContent?key=${GEMINI_API_KEY}`, body `{ contents:[{parts:[{text:prompt}]}], generationConfig:{ responseModalities:["IMAGE"] } }`. Map `candidates[0].content.parts[].inlineData.data`→imageBase64; `usageMetadata`→usage.
Integration skip guard: `const RUN = process.env.RUN_AI_INTEGRATION === "1" && !!process.env.OLLAMA_CLOUD_API_KEY; (RUN ? describe : describe.skip)(...)` (Gemini variant uses `GEMINI_API_KEY`).

## Risks & design decisions the planner made (for CHECKER scrutiny)
1. **SDK-vs-fetch: plain `fetch`, zero new deps (RECOMMENDED).** Both surfaces are simple HTTP+JSON; `openai`/`@google/genai` add heavy transitive deps for two endpoints and would violate CLAUDE §2 rule 7 without strong justification. AbortController gives timeouts; Node ≥22 has global `fetch`. Trade-off: we hand-map response shapes (covered by unit specs). Checker: confirm no SDK import and no lockfile dep beyond `zod`/workspace.
2. **Selector envs are explicit enums** (`AI_TEXT_PROVIDER=ollama|fake`, `AI_VISION_PROVIDER=ollama|fake`, `AI_IMAGE_PROVIDER=gemini|fake`), defaulting to the real providers, `fake` for tests/CI — chosen over NODE_ENV-driven magic for testability and an explicit CI knob. Added to `aiEnvSchema` + `.env.example`.
3. **Cost model = plumbing, not real prices.** `computeUsage` records tokens+latency and computes `costMicroUsd` via a pluggable `CostRates` defaulting to 0 (Ollama Cloud pricing varies; re-baselined later per AI_PROVIDERS §4). The AC "cost/latency capture unit-tested" is satisfied by testing the capture path, not real tariffs. No cost-rate env vars added this task.
4. **`ai` env schema lives in `packages/ai` (its own slice via `defineEnv`), not `packages/config`** — mirrors api's `env.schema.ts` pattern; keeps provider env colocated with the seam. Defaults use the existing fake example values so keyless CI/typecheck stays green.
5. **Gemini endpoint/body shape is the biggest external-contract assumption** (`v1beta/.../:generateContent` with `responseModalities:["IMAGE"]` + `inlineData.data`). The configured `GEMINI_IMAGE_MODEL` may instead need an Imagen `:predict` shape. Impl is kept thin and fully mockable; the (CI-skipped) integration spec is the real-shape check. If the real model needs a different path, that is a one-file edit inside this seam. Flagged as the top risk to re-verify at T034/T052.
6. **No `tsconfig.base.json` path alias for `@pawcareright/ai` and real impls not re-exported from the barrel.** types/api-client have aliases because they are consumed; nothing imports `@pawcareright/ai` yet, so adding the alias now is speculative (CLAUDE §2 rule 2) and would edit `packages/config` (out of scope). The alias + barrel surface are added by the first consumer task (T034). Consequence: this package is self-contained/tested but not yet wired — intended. Checker: absence of the alias is deliberate, not an omission.
7. **Integration "how to run locally" is a header comment in each `*.integration.spec.ts`, not a new README** — keeps the file list tight; no doc file needed by the ACs.
