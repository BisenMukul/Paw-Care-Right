## Deviations assessed
1. **`HttpException(_, 402)` vs PaymentRequiredException:** correct — `STATUS_TO_CODE` maps 402 → `PAYMENT_REQUIRED` (`all-exceptions.filter.ts:17`), mirrors the 429 pattern. Sound.
2. **`nextCursor = last returned item's id` (not the plan's "extra row's id"):** **the executor is right; this is a correct fix.** Prisma `cursor` is inclusive and `skip:1` skips the cursor row, so a page resumes *after* the cursor. Trace (3 rows desc `[a,b,c]`, limit 2): page1 returns `[a,b]`, `nextCursor="b"`; page2 (`cursor=b, skip:1`) starts after `b` → returns `[c]`. The plan's literal "extra row id `c`" + skip:1 would start after `c`, **dropping `c`** (a gap). e2e `:344-368` proves page1=`[newest,mid]`, page2=`[oldest]`, final `nextCursor=null` — no gap/duplicate. Unit `:360-382` pins semantics + the `cursor/skip/take` passthrough.
3. **blood_in_urine-alone → `highest===null`:** honest and correct against `rules-table.ts` — there is no standalone blood-in-urine rule; `urinary-obstruction-signs-cat` (rule 19) requires straining **AND** blood. The combo test (`mapper.spec.ts:104-113`) sets both signs and asserts `highest !== null` (fires rule 3 `urinary-blockage-cat` and/or rule 19). Matches the table.
4. **P2002 loser re-enqueues with `jobId=checkId`:** BullMQ no-ops on an existing jobId, so the duplicate `add` is harmless. Sane per D2/queue contract.
