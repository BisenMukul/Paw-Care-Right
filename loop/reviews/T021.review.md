- `pets.service.ts` — EVERY query carries `householdId` + `deletedAt: null`; no bare `findUnique(id)` for reads:
  - `findAll` `:58` — `where: { householdId, deletedAt: null }`
  - `findOne` `:67` — `where: { id, householdId, deletedAt: null }`
  - `update` `:79` — loads scoped `findFirst where { id, householdId, deletedAt: null }` (404 at `:83`) before the `update where: { id }` at `:105` — the `{ id }` write is reached only after scope+soft-delete are proven, so it cannot cross households.
  - `remove` `:124` — same scoped `findFirst` guard (404 at `:128`) before the `update where: { id }` soft-delete at `:131`.
