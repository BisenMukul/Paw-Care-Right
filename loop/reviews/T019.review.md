## AC verification (code-quoted)

### AC1 — 401 → single refresh → retry original once
`packages/api-client/src/client.ts`:
- Interceptor gated on config: `if (first.response.status === 401 && config.refreshSession)` (L119) — bare clients (no `refreshSession`) keep T010 behavior verbatim.
- Single-flight: closure `let refreshInFlight` (L54) + `refreshOnce()` (L100–110) sets `refreshInFlight` once, concurrent 401s `await` the same promise, cleared in `.finally` (L105–107).
- Retry carries NEW token via explicit override: `attempt(path, init, token)` (L127) → `headers.set("Authorization", \`Bearer ${authOverride}\`)` (L69–70), NOT re-read via `getAuthToken`.
- Exactly ONE retry: on retried non-ok → `onSessionExpired` + throw (L133–134), no loop back into refresh.
- Specs (`client.spec.ts` +123, additions-only): "retries once after a 401 with the refreshed token" asserts fetch×2, refresh×1, `Authorization: Bearer new-token` on 2nd call; "dedupes concurrent 401s into one refresh" asserts refresh×1 across two concurrent requests (fetch×4); "calls onSessionExpired on a second 401 after retry (no loop)" asserts refresh×1, onSessionExpired×1, fetch×2 (no 3rd fetch).

### AC2 — refresh failure logs out
- `client.ts` L122–124: `token === null` → `await config.onSessionExpired?.()` + throw normalized 401. `refreshOnce` `.catch(() => null)` (L104) folds a thrown refresh into null, so both null-return and throw hit this path. Specs "…when refresh returns null" and "…when refresh throws" assert onSessionExpired×1, fetch×1 (no retry), rethrown `UNAUTHORIZED`/401.
- Mobile local-only sign-out: `auth-store.ts` `sessionExpired()` (L85–93) = `clearTokens()` + `set({ status: "signedOut", ... })`. NO `authApi.logout` call. Test "clears SecureStore and sets signedOut without calling logout" asserts `mockedAuthApi.logout` **not** called.
- Interceptor re-entry impossible: `apps/mobile/src/api/client.ts` — `authClient` (L10–13) is BARE (no `refreshSession`/`onSessionExpired`); `auth-api.ts` routes refresh/logout exclusively through `authClient` (L28–33). A 401 on the refresh endpoint cannot re-enter the interceptor. `apiClient` (L21–26) delegates `refreshSession`/`onSessionExpired` to the store.
