## BLOCKING DEFECT — `.github/workflows/ci.yml` was NOT modified

`git diff .github/workflows/ci.yml` is **empty**; `git status` shows no change to it.
The plan's file list (line 41) and Risk **R6** mandate the sole sanctioned cross-task
edit: a "Start MinIO" step in the `build` job before `pnpm test`. The task card given to
the checker likewise lists ci.yml as MODIFIED ("the sanctioned MinIO docker-run step").

Consequence (verified against the current ci.yml, which provisions only `postgres` +
`redis` services, no MinIO): the acceptance-critical `test/photos.e2e-spec.ts` has **no
skip guard** — its `beforeAll` unconditionally calls `storage.ensureBucket()` (L60) and
opens `QueueEvents`. In CI there is no `localhost:9000`, so `ensureBucket()` will throw
(ECONNREFUSED) → `beforeAll` fails → the whole suite fails → **`pnpm test` goes red in
CI**. R6 explicitly chose the no-skip-guard design *specifically so the suite cannot
self-skip* — which makes the omitted CI step a hard CI breakage, not a silent gap.

This regresses the DoD §8 `pnpm test` gate and violates CLAUDE.md §5 ("If a command in
this list breaks, fixing it is an implicit P0 acceptance criterion of the current task").
Locally everything is green only because MinIO is up via compose; the delivered change
breaks the CI `pnpm test` gate. A required, planned file was omitted and its omission
actively breaks CI.

Fix: add the R6 "Start MinIO" step (docker run minio + health wait) to the `build` job
before `pnpm test`, and add the dependency + ci.yml justifications to `loop/journal.md`.
